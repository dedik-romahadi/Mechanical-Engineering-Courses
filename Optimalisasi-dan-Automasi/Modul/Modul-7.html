Bagian 01

Mengapa Domain Frekuensi:
Sidik Jari Periodik yang Tersembunyi di Domain Waktu

Fitur domain waktu (Pertemuan 6) sangat baik untuk menangkap energi, impulsivitas, dan distribusi sinyal — tetapi kerap buta terhadap kerusakan yang menghasilkan frekuensi spesifik: bearing fault, gear-mesh fault, blade-pass anomaly. Solusinya: pindah ke domain frekuensi lewat Discrete Fourier Transform (DFT) yang dieksekusi efisien sebagai FFT. Spektrum yang dihasilkan menunjukkan secara eksplisit "berapa energi pada frekuensi berapa" — peak diskrit di sini = sumber periodik di mesin.

Lima Konsep Inti Domain Frekuensi
DFT  ·  FFT  ·  PSD  ·  Frekuensi karakteristik (BPFO/BPFI/BSF/FTF)  ·  Envelope analysis
🔍

Lihat Periodicity yang Tak Tampak

Sebuah sinyal vibrasi dengan amplitudo kecil pada 177 Hz hampir mustahil dilihat di plot waktu — tertutupi noise. Tetapi di spektrum FFT, ia muncul sebagai peak diskrit setinggi gunung. Inilah keunggulan domain frekuensi: memisahkan komponen periodik dari noise broadband bahkan saat SNR domain waktu rendah.

x(t) → X(f) = FFT[x]
🧮

FFT: O(N log N), Bukan O(N²)

DFT naif butuh O(N²) komputasi. FFT Cooley–Tukey (1965) memanfaatkan struktur radix-2 untuk turun ke O(N log N) — perbedaan 100×–1000× untuk N besar. Inilah mengapa FFT 8192-point bisa dijalankan real-time di edge device. Library: numpy.fft.fft, scipy.fft.fft, dan scipy.signal.welch untuk PSD.

FFT: O(N log N)
⚙️

Diagnostik Spesifik per Kerusakan

Setiap kerusakan mesin punya tanda tangan frekuensi unik. Bearing outer-race fault → peak di BPFO. Gear chipped tooth → peak di frekuensi mesh ± gear-shaft frequency. Blade unbalance → peak di blade-pass frequency. Domain waktu hanya bisa melihat "ada sesuatu yang salah"; domain frekuensi memberitahu komponen mana yang rusak.

peak @ BPFO → outer-race
📡

Komplemen, Bukan Pengganti

Domain frekuensi tidak menggantikan domain waktu — keduanya saling melengkapi. RMS & kurtosis (waktu) tetap dipakai untuk severity scoring; peak FFT & band power (frekuensi) untuk fault classification. Best practice industri: 12 fitur waktu + 12 fitur frekuensi = 24-dim feature vector, langsung di-feed ke ML classifier (Pertemuan 13).

[φ_t, φ_f] ∈ ℝ^24 → ML
Posisi Pertemuan 7 dalam Alur Data Pipeline:
P3 (kumpul & visualisasi) → P4 (pembersihan) → P5 (transformasi) → P6 (fitur domain waktu) → P7 (Anda di sini — domain frekuensi: FFT, PSD, BPFO/BPFI, envelope) → P9–13 (modeling). Pertemuan ini melengkapi feature engineering: gabungan 12 fitur waktu + 12 fitur frekuensi memberi feature vector kaya yang dipakai sebagai input ke model klasifikasi.
💡 Prinsip Utama: Sebelum FFT, sinyal harus stasioner secara lokal di window analisis. Untuk sinyal non-stasioner pakai STFT/spectrogram (lihat Bagian 06). Selalu (1) buang DC offset sebelum FFT; (2) terapkan window function (Hann, Hamming, Blackman) untuk menekan spectral leakage; (3) gunakan Welch's method untuk PSD yang lebih halus dan robust dibanding periodogram tunggal.
Bagian 02

DFT & FFT:
Dari Sampel Diskrit ke Spektrum Frekuensi

Discrete Fourier Transform (DFT) memetakan N sampel waktu x[n] menjadi N koefisien frekuensi X[k] kompleks. Fast Fourier Transform (FFT) adalah algoritma efisien untuk DFT — sama hasilnya, jauh lebih cepat. Hasil FFT memberi tahu amplitudo dan fase tiap komponen frekuensi pada grid diskrit dengan resolusi Δf = fs/N.

Empat Persamaan Fundamental Domain Frekuensi
X[k] = Σn=0..N−1 x[n]·e−j2πkn/N  ·  fk = k·fs/N  ·  Δf = fs/N  ·  |X[k]| = √(Re² + Im²)
🔢

DFT (Discrete Fourier Transform)

Definisi formal: X[k] = Σ x[n]·e^(−j2πkn/N), untuk k = 0..N−1. Setiap koefisien X[k] adalah bilangan kompleks — magnitude-nya = energi pada frekuensi f_k = k·fs/N, sudut-nya = fase. Komputasi langsung butuh N² operasi kompleks. Untuk N = 8192, itu ≈ 67 juta operasi per FFT — terlalu lambat untuk real-time.

DFT: O(N²)
⚡

FFT (Fast Fourier Transform)

Algoritma Cooley–Tukey (1965) memecah DFT N-titik menjadi rekursi pada DFT N/2-titik bersama "twiddle factors". Hasilnya O(N log N). Untuk N = 8192: hanya ≈ 100 ribu operasi (660× lebih cepat dari DFT naif). Implementasi modern: numpy.fft.rfft (real-input, lebih cepat 2×) atau scipy.fft.fft.

FFT: O(N log N)
📐

Frekuensi Resolusi (Δf)

Resolusi frekuensi: Δf = fs / N. Sampling 25 kHz dengan N = 8192 → Δf ≈ 3.05 Hz. Untuk membedakan dua peak yang berdekatan (misal BPFO 177 Hz vs 180 Hz line frequency), Δf harus lebih kecil. Solusinya: perbesar N (window panjang) — tapi membutuhkan asumsi sinyal tetap stasioner sepanjang window. Trade-off klasik: resolusi vs stationarity.

Δf = fs / N
🪟

Window Function (Hann, Hamming, Blackman)

FFT mengasumsikan sinyal periodik di window. Jika sinyal real tidak persis terkemas di window N, terjadi spectral leakage — energi peak menyebar ke bin tetangga. Solusi: kalikan sinyal dengan window function sebelum FFT. Hann (umum), Hamming, Blackman (leakage paling kecil), atau flat-top (untuk akurasi amplitudo). Selalu pakai window — rectangular hampir selalu jelek.

x_w[n] = w[n]·x[n]
One-Sided vs Two-Sided Spectrum
Untuk sinyal real, X[k] simetris konjugat: X[N−k] = X[k]*. Setengah spektrum sudah cukup. Pakai numpy.fft.rfft (one-sided) — output panjangnya N/2+1, hemat memori & komputasi. Untuk konversi amplitudo "satuan asli": kalikan |X[k]| dengan 2/N untuk semua k kecuali DC dan Nyquist (k=0 dan k=N/2 dikalikan 1/N). Tanpa scaling ini, amplitudo peak di spektrum tidak sama dengan amplitudo sinusoidal di domain waktu.
💡 Aturan Praktis Pemilihan N: (1) Untuk monitoring umum (RMS, broadband), N = 1024–2048 cukup. (2) Untuk identifikasi BPFO/BPFI yang dekat dengan harmoni line frequency, N = 8192–32768 (Δf < 1 Hz). (3) N harus pangkat 2 untuk FFT optimal — gunakan N = next_pow2(target_N). (4) Sinyal pendek bisa di-zero-pad ke pangkat 2 — ini memberi interpolasi spektral, bukan resolusi tambahan; resolusi sejati tetap fs/N_asli.
Bagian 03

Magnitude Spectrum & Power Spectral Density:
"Berapa Energi pada Frekuensi Berapa"

Hasil mentah FFT adalah deretan koefisien kompleks. Untuk visualisasi dan ekstraksi fitur, kita pakai magnitude spectrum (akar Re²+Im²) atau lebih baik Power Spectral Density (PSD) yang ter-normalisasi terhadap fs dan window. PSD memberi nilai dengan satuan amplitudo² / Hz — tidak tergantung panjang window, ideal untuk membandingkan spektrum dari segmen panjang berbeda.

Empat Definisi Spektrum
|X[k]| (magnitude)  ·  |X[k]|² (power)  ·  PSD = |X[k]|² / (fs·N)  ·  Welch PSD = average over overlapping segments
📊

Magnitude Spectrum |X[k]|

Plot |X[k]| vs f_k = k·fs/N. Peak diskrit menunjukkan komponen periodik. Untuk sinyal real, plot hanya bagian one-sided (k = 0..N/2). Magnitude di-skalakan dengan 2/N agar peak amplitudo cocok dengan amplitudo sinusoidal asli (kecuali DC dan Nyquist yang dikalikan 1/N).

A_k = (2/N)·|X[k]|
⚡

Periodogram (Power Spectrum)

Estimasi power spectrum sederhana: P[k] = |X[k]|² / (fs·N) — disebut periodogram. Cepat dihitung tetapi variansi tinggi (estimator inkonsisten secara statistik): noise spectrum sangat berisik. Tidak cocok untuk diagnostik fault yang butuh peak SNR baik. Solusinya: average banyak periodogram → Welch's method.

P[k] = |X[k]|² / (fs·N)
🧮

Welch's Method (Recommended)

Bagi sinyal panjang menjadi segmen overlap (50% umum), terapkan window di tiap segmen, hitung periodogram, lalu rata-rata. Hasil: PSD jauh lebih halus (variansi turun ~2/K untuk K segmen). Library: scipy.signal.welch(x, fs, nperseg=2048, noverlap=1024, window='hann') — standar industri untuk vibration analysis.

PSD_W = (1/K)·ΣP_i
📏

Parseval's Theorem (Cross-Check)

Energi total sinyal sama di kedua domain: Σ|x[n]|² = (1/N)·Σ|X[k]|². Untuk PSD: integral PSD over frekuensi = variance sinyal di waktu. Cross-check wajib implementasi: hitung RMS dari sinyal waktu, hitung √(Σ PSD·Δf) — keduanya harus sama (dalam toleransi numerik). Jika tidak sama → ada bug di scaling atau window.

Plot linear menyembunyikan peak kecil; plot dB menonjolkannya. Praktik: monitoring trend pakai linear scale (gampang lihat absolute change), diagnostik fault pakai dB.

Deviasi dari rasio-rasio ini menandakan sinyal non-Gaussian — biasanya karena ada komponen impulsif atau periodik. Sangat berguna untuk quick health check sebelum menghitung fitur orde tinggi yang lebih mahal.
Bagian 04

Frekuensi Karakteristik Bearing:
BPFO, BPFI, BSF, FTF — Sidik Jari Kerusakan

Inilah inti diagnostik domain frekuensi: tiap kerusakan bearing menghasilkan frekuensi karakteristik unik yang dihitung dari geometri (jumlah bola, diameter bola, pitch diameter, contact angle) dan kecepatan putar. Peak diskrit di frekuensi-frekuensi ini = bukti definitif lokasi kerusakan.

Empat Frekuensi Karakteristik Bearing (n bola, d ball, D pitch, α contact angle, fr shaft speed)
BPFO = (n/2)·fr·(1 − (d/D)·cosα)  ·  BPFI = (n/2)·fr·(1 + (d/D)·cosα)  ·  BSF = (D/2d)·fr·(1 − (d/D)²·cos²α)  ·  FTF = (fr/2)·(1 − (d/D)·cosα)
⭕

BPFO — Outer-race Defect

Ball Pass Frequency, Outer race: BPFO = (n/2)·fr·(1−(d/D)·cosα). Bearing 6205 (n=9, d=7.94 mm, D=39 mm, α=0°) pada motor 1480 rpm (fr ≈ 24.67 Hz): BPFO ≈ 88.6 Hz. Saat outer race rusak (paling umum: spalling lokal), tiap bola yang lewat defect menghasilkan impak. Peak diskrit muncul di BPFO + harmoni (2·BPFO, 3·BPFO).

BPFO ≈ (n/2)·fr·(1−d/D·cosα)
🔵

BPFI — Inner-race Defect

Ball Pass Frequency, Inner race: BPFI = (n/2)·fr·(1+(d/D)·cosα). Untuk 6205 di 1480 rpm: BPFI ≈ 133.3 Hz. Selalu > BPFO. Ciri khas inner-race fault: peak BPFI dimodulasi oleh fr (karena inner race berputar bersama shaft) — terlihat sebagai sideband di BPFI ± fr di spektrum envelope.

BPFI > BPFO selalu
⚪

BSF — Ball / Roller Defect

Ball Spin Frequency: BSF = (D/2d)·fr·(1 − (d/D)²·cos²α). Untuk 6205 di 1480 rpm: BSF ≈ 56.9 Hz. Saat bola sendiri rusak (spalling pada permukaan bola), defect masuk-keluar zona beban tiap setengah putaran — peak BSF muncul tetapi sering bersama harmoni 2·BSF. Defect bola lebih sulit dideteksi dibanding race fault karena posisinya berputar bersama cage.

BSF ≪ BPFO biasanya
🌀

FTF — Cage Defect (Fundamental Train Frequency)

Frekuensi cage: FTF = (fr/2)·(1−(d/D)·cosα). Untuk 6205 di 1480 rpm: FTF ≈ 9.85 Hz. Selalu < fr/2. Peak di FTF muncul saat cage retak / aus, biasanya akhir umur bearing. FTF rendah, cocok dideteksi dengan FFT resolusi tinggi (Δf < 0.5 Hz). Sering ada sebagai modulasi sideband di sekitar BPFO/BPFI.

FTF < fr/2 selalu
🔁

Harmoni & Sideband

Bearing fault jarang muncul sebagai peak tunggal — ada harmoni (2×, 3×, 4× frekuensi) dan sideband. Outer race: harmoni dominan. Inner race: harmoni + sideband ±fr. Cage: sideband di sekitar BPFO/BPFI ±FTF. Pola ini diandalkan untuk klasifikasi fault — tidak hanya ada peak, tapi pola peak.

k·BPFI ± m·fr
🎵

Envelope Analysis (HFRT)

Bearing fault menghasilkan resonansi struktural di 5–20 kHz yang dimodulasi oleh frekuensi karakteristik. Trik: (1) bandpass ke band resonansi, (2) hitung envelope (Hilbert transform), (3) FFT envelope. Hasilnya: peak BPFO/BPFI/BSF tampak sangat jelas, jauh lebih bersih daripada FFT raw signal. Ini adalah teknik standar industri (SKF Multilog, Bently Nevada).

⚠️

Slip dalam Realita

Frekuensi BPFO/BPFI di atas dihitung dengan asumsi no-slip — bola rolling murni. Kenyataannya bola sedikit slipping (terutama saat beban radial rendah), sehingga frekuensi observasi 1–3% lebih rendah dari prediksi teoretis. Saat mencari peak di spektrum, beri toleransi ±5% di sekitar nilai prediksi. Peak yang konsisten muncul dalam toleransi ini, plus harmoni / sideband, = bukti diagnostik kuat.

Bagian 05

Analogi
Kehidupan Sehari-hari

Konsep FFT, spektrum, dan envelope analysis sudah ada di sekitar Anda — di pengaturan equalizer audio, deteksi nada lagu di Shazam, hingga pemeriksaan irama jantung modern. Enam analogi di bawah ini menjelaskan kenapa melihat sinyal di domain frekuensi sering jauh lebih informatif daripada di domain waktu.

🎵 Analogi 1 — Equalizer Audio & Spektrum Frekuensi Live Saat Anda buka equalizer di Spotify atau aplikasi player, deretan bar bergerak yang Anda lihat adalah magnitude spectrum dari sinyal audio yang sedang main. Bar di kiri = bass (50–200 Hz), tengah = vocal (300–3000 Hz), kanan = treble/cymbals (5–15 kHz). Equalizer real-time menghitung FFT pada window pendek (~20 ms) dan mem-plot magnitude. Sama persis seperti monitoring vibrasi industri: spektrum FFT bergerak menampilkan "isi frekuensi" sinyal mesin secara live, dan engineer mencari peak abnormal seperti sound engineer mencari resonansi yang dominan.
🎶 Analogi 2 — Shazam & Constellation Map Shazam mengenali lagu dalam 5 detik dengan trik FFT cerdas: (1) STFT sinyal audio Anda, (2) ambil koordinat (frekuensi, waktu) dari peak terkuat — disebut constellation map, (3) hash pasangan peak dan bandingkan dengan database. Database lagu di-precompute dengan algoritma yang sama. Pencocokan terjadi di domain frekuensi, bukan waktu — itu sebabnya Shazam tahan terhadap noise restoran dan distorsi codec. Diagnostik bearing pakai prinsip serupa: peak diskrit di BPFO/BPFI = sidik jari kerusakan, dicocokkan dengan tabel referensi.
📡 Analogi 3 — Radio Tuning & Selektivitas Frekuensi Radio FM bekerja karena tiap stasiun mengirim di frekuensi spesifik (FM 88.0, 89.5, 100.7 MHz). Tuner pakai bandpass filter sempit untuk pilih satu frekuensi dan tolak lainnya. Inilah kenapa Anda bisa dengar satu stasiun dengan jelas tanpa noise dari stasiun tetangga. Sama prinsipnya dengan envelope analysis: bearing fault menumpang resonansi di band 5–20 kHz; kita "tune in" ke band itu dengan bandpass, lalu demodulasi untuk dapat informasi BPFO/BPFI yang ter-modulasi di sana.
🌈 Analogi 4 — Prisma Newton & Decompose Cahaya Putih Newton menunjukkan: cahaya putih = superposisi semua warna. Prisma "memecahnya" jadi spektrum visible (merah → ungu). Mata kita menerima campuran tetapi prisma membongkar isinya. FFT melakukan hal yang persis sama untuk sinyal vibrasi: sinyal time-domain = superposisi sinusoidal di banyak frekuensi, dan FFT memecahnya menjadi spektrum yang menampilkan "berapa energi pada frekuensi berapa". Newton butuh prisma kaca; engineer pakai numpy.fft.fft.
❤️ Analogi 5 — Heart Rate Variability (HRV) Spectrum Cardiologist modern menganalisis irama jantung di domain frekuensi juga: HRV spectrum dibagi LF (0.04–0.15 Hz) dan HF (0.15–0.4 Hz). Rasio LF/HF menunjukkan keseimbangan saraf simpatis/parasimpatis. Dua orang dengan rata-rata BPM sama bisa punya HRV spectrum sangat berbeda — yang stress punya LF/HF tinggi. Sama dengan vibrasi: dua mesin dengan RMS sama bisa punya spektrum berbeda total — yang sehat punya broadband datar, yang fault punya peak diskrit di BPFO/BPFI.
🎙️ Analogi 6 — Spektrogram Suara & Voice Print Forensic phonetician identifikasi pembicara dari spektrogram — STFT dari rekaman suara. Pola formant (F1, F2, F3) di band 200–3500 Hz unik per orang seperti sidik jari. Spektrogram mengubah waktu vs amplitudo (1D) menjadi waktu vs frekuensi vs intensitas (2D heatmap). Untuk diagnostik mesin yang sinyalnya tidak stasioner (run-up motor, gear shift), spektrogram menggantikan FFT tunggal: bagaimana frekuensi berubah seiring waktu.
💡 Pola yang Sama di Semua Analogi: Setiap kasus melakukan transformasi waktu → frekuensi untuk mengungkap struktur yang tidak terlihat di domain asli. Equalizer, Shazam, prisma, HRV, spektrogram — semua memanfaatkan fakta bahwa banyak fenomena di alam periodik dan periodicity itu paling jelas di domain frekuensi. Untuk vibrasi mesin: kerusakan periodik (bearing, gear) = peak diskrit di FFT; kerusakan stokastik (lubrikasi, noise) = broadband di FFT. Domain frekuensi membongkar perbedaan ini.

Animasi 2 — Bearing Fault Frequencies: BPFO, BPFI, BSF, FTF di Spektrum
📊 Cara Membaca: Atas = sinyal vibrasi bearing dengan injeksi periodic impulses (impak periodic dari fault). Bawah = magnitude spectrum dengan peak pada frekuensi karakteristik: BPFO (Ball Pass Frequency Outer), BPFI (inner race), BSF (ball spin), FTF (fundamental train). Tiap fault punya signature peak berbeda.
Coba: (1) Frekuensi impak rendah (~mendekati BPFO ~3.6× shaft RPM) → peak di posisi BPFO. (2) Tambah amplitudo impak → peak height naik & harmonik (2×, 3× BPFO) muncul. (3) Frekuensi mendekati BPFI (~5.4× RPM) → peak shift, sidebands muncul (modulasi shaft RPM). Pelajaran: frekuensi peak posisi = jenis fault; harmonik & sidebands = severity. Wajib hitung BPFO/BPFI dari geometri bearing (datasheet SKF/FAG) sebelum interpretasi spektrum.
Animasi 3 — Magnitude Spectrum vs PSD: Linear vs Power Density
📊 Cara Membaca: Atas = magnitude spectrum (linear amplitude mm/s vs frekuensi Hz). Bawah = Power Spectral Density / PSD (power per Hz, mm²/s²/Hz vs Hz). Slider kontrol jumlah peak frekuensi & level noise. PSD memperlihatkan distribusi energi, magnitude spectrum memperlihatkan amplitudo komponen.
Coba: (1) Single peak → magnitude amplitude tinggi, PSD menunjukkan power di band sempit. (2) Multi peak → magnitude menampilkan tiap peak, PSD menampilkan sebagai delta function thin. (3) Pure noise → magnitude flat tetapi noisy, PSD smooth karena averaging. Kapan pakai mana: Magnitude untuk identifikasi peak frekuensi (BPFO, harmonics). PSD untuk band energy comparison (energi 0–500 Hz vs 500–1000 Hz), random vibration analysis, dan compliance test (MIL-STD 810, IEC 61373).
Animasi 4 — Spectrogram (STFT): Time-Frequency Map untuk Sinyal Non-Stationary
📊 Cara Membaca: Spectrogram = STFT (Short-Time Fourier Transform) — series FFT di window sliding di sinyal. Sumbu X = waktu, Y = frekuensi, warna = amplitude. Visualisasi bagaimana spektrum berubah seiring waktu. FFT klasik asumsikan sinyal stasioner (statistik konstan). Untuk sinyal non-stationary (run-up/run-down turbine, impak transient, frequency sweep), spectrogram membuka informasi yang FFT global tidak bisa kasih.
Coba: (1) Slider posisi waktu rendah (early lifetime) → spectrogram menampilkan peak di shaft frequency saja. (2) Slider tengah (early fault) → peak baru muncul di BPFO + harmonik. (3) Slider akhir (severe fault) → spectrogram chaos dengan banyak peak + harmonik + broadband. Tools: scipy.signal.spectrogram(x, fs, nperseg=256). Wavelet transform = alternative untuk analisis time-frequency dengan resolusi adaptif (lebih detail di transient pendek).

Jebakan Praktis FFT & PSD

(1) Lupa window function → spectral leakage; selalu pakai Hann minimal. (2) Salah scaling magnitude — kalikan 2/N untuk amplitudo asli (kecuali DC dan Nyquist). (3) FFT sinyal non-stasioner (run-up motor) → peak smear; pakai STFT atau order tracking. (4) Resolusi terlalu kasar untuk pisahkan BPFO dari line frequency 50 Hz harmoni → perbesar N atau pakai zoom-FFT. (5) Aliasing jika fs < 2·f_max → lakukan anti-aliasing filter analog sebelum ADC. (6) Periodogram saja untuk diagnostik → variansi tinggi; pakai Welch.

# Komponen utama: rotasi shaft 30 Hz (1800 rpm) + harmonik 2× & 3×
f_rot = 30                                # Hz
sig = (1.0*np.sin(2*np.pi*f_rot*t)
       + 0.4*np.sin(2*np.pi*2*f_rot*t)
       + 0.2*np.sin(2*np.pi*3*f_rot*t))
sig += 0.3*np.random.randn(N)               # background noise

# Injeksi impak BPFO (Ball Pass Frequency Outer race) ~ 107.7 Hz untuk 6205 di 30 Hz shaft
# Setiap impak: damped sinusoid carrier ~3 kHz (resonansi housing bearing)
f_bpfo = 107.7
period_samples = int(fs / f_bpfo)
for i in range(15, N, period_samples):

Cell 4 — Fungsi extract_features_freq() + Envelope Analysis (Bandpass + Hilbert + FFT) Python 
from scipy.signal import butter, sosfiltfilt, hilbert
from scipy.stats import entropy as sp_entropy

def extract_features_freq(x, fs):
    """Ekstrak 12 fitur domain frekuensi dari sinyal x. Return dict."""
    N      = len(x)
    X      = rfft(x)
    f      = rfftfreq(N, d=1/fs)
    P      = (np.abs(X)**2) / N         # power spectrum
    Psum   = P.sum() + 1e-12
    p_norm = P / Psum                  # prob distribution di freq

    peak_freq = f[np.argmax(P)]                    # dominant frequency
    mean_freq = np.sum(f * p_norm)                  # spectral centroid
    var_freq  = np.sum((f - mean_freq)**2 * p_norm)   # spectral variance
    sk_spec   = np.sum((f - mean_freq)**3 * p_norm) / (var_freq**1.5 + 1e-12)
    ku_spec   = np.sum((f - mean_freq)**4 * p_norm) / (var_freq**2 + 1e-12)
    sp_ent    = sp_entropy(p_norm + 1e-12)             # Shannon entropy spektral

    # Band powers: low (0–200), mid (200–1000), high (1000–5000)
    bp_low  = P[(f >= 0)    & (f < 200)].sum()
    bp_mid  = P[(f >= 200)  & (f < 1000)].sum()
    bp_high = P[(f >= 1000) & (f < 5000)].sum()

    return {
        'peak_freq': peak_freq, 'mean_freq': mean_freq,
        'rms_freq': np.sqrt(var_freq),
        'spectral_skewness': sk_spec,
        'spectral_kurtosis': ku_spec,
        'spectral_entropy': sp_ent,
        'total_power': Psum,
        'band_power_low': bp_low,
        'band_power_mid': bp_mid,
        'band_power_high': bp_high,
        'ratio_high_to_low': bp_high / (bp_low + 1e-12),
        'parseval_check': Psum,                 # ≈ N · mean(x²)
    }

feat = extract_features_freq(sig, fs)
print("═══ 12 Fitur Domain Frekuensi ═══")
for k, v in feat.items():
    print(f"  {k:22s} = {v:12.4f}")

# --- ENVELOPE ANALYSIS: bandpass [2500–3500] Hz → Hilbert → FFT envelope ---
sos     = butter(4, [2500, 3500], btype='bandpass', fs=fs, output='sos')
sig_bp  = sosfiltfilt(sos, sig)                # isolasi resonansi bearing housing
env     = np.abs(hilbert(sig_bp))              # envelope (instantaneous amplitude)
env    -= env.mean()                          # DC removal sebelum FFT
Env_f   = (2.0/N) * np.abs(rfft(env))
freqs_e = rfftfreq(N, d=1/fs)
mask_e  = freqs_e < 300

ax.set_title('Envelope Spectrum — peak di BPFO mengkonfirmasi outer-race defect')

Diagnosis: Apakah peak 88 Hz pada spektrum + envelope spectrum mengkonfirmasi outer-race defect?

›

Peak struktural pompa (blade-pass) — abaikan saja

Misalignment shaft — biasanya muncul di 2× fr

02

Mengapa envelope analysis lebih sensitif untuk early bearing fault dibanding FFT raw? Jelaskan dari sudut pandang signal processing.

›

Pada early bearing fault, peak BPFO seringkali tertutupi oleh komponen dominan low-frequency (1×fr, 2×fr, blade-pass, dll). Envelope analysis (bandpass band resonansi 2–4 kHz → Hilbert transform → FFT envelope) men-demodulasi impulse train yang terkubur di band high-frequency. Jelaskan secara matematis dan visual mengapa pendekatan ini superior.

💬 Petunjuk: bearing impak menghasilkan model "impulse train × resonance carrier" — energi periodik di rate BPFO termodulasi pada frekuensi resonansi housing (~beberapa kHz). FFT raw melihat resonansi sebagai broadband bump yang sulit di-isolasi. Bandpass + Hilbert mengekstrak amplitudo termodulasi (envelope), dan FFT envelope mengembalikan rate BPFO sebagai peak tajam.
QUICK CHECK — Pendekatan terbaik untuk identifikasi early bearing fault adalah...
FFT raw saja — cari peak tertinggi langsung
Time-domain features (RMS, kurtosis) — paling cepat & murah
Envelope analysis (bandpass + Hilbert + FFT) di band resonansi yang dipilih via kurtogram
Pakai sinyal raw mentah, biar AI yang belajar
✅ Tepat! Envelope analysis adalah gold standard untuk diagnosa bearing (Randall & Antoni, 2011). Pipeline: (1) Kurtogram untuk pilih band resonansi optimal (band dengan spectral kurtosis tertinggi); (2) Bandpass filter untuk isolasi resonansi yang dimodulasi; (3) Hilbert transform untuk ekstrak envelope (amplitudo instantan); (4) FFT envelope untuk reveal rate impak (BPFO/BPFI/FTF). FFT raw saja sering miss early fault karena energi BPFO masih kecil dibanding fr/2fr/3fr. Time-domain features (RMS, kurtosis) memberi sinyal "ada masalah" tetapi tidak bisa identifikasi tipe defect — envelope spectrum bisa: peak BPFO → outer race, peak BPFI + sideband ±fr → inner race, peak FTF → cage.
❌ FFT raw rentan terhadap dominansi komponen low-frequency (fr, blade-pass, line frequency); peak BPFO sering tenggelam. Time-domain features tidak bisa membedakan jenis defect. ML "blackbox" tanpa preprocessing yang tepat butuh ribuan label data — tidak realistis untuk most industrial setting. Envelope analysis tetap juara karena memanfaatkan struktur fisik fenomena bearing fault.
