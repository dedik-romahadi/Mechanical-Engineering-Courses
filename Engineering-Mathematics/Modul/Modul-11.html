Bagian 01

Penerapan Laplace —
Toolkit Engineering Lanjut

Pertemuan 10 mengenalkan dasar Transformasi Laplace: definisi, tabel fungsi dasar, sifat-sifat, transformasi turunan, dan workflow solve PD. Pertemuan 11 melengkapi toolkit dengan 5 konsep aplikasi inti yang membuat Laplace tak tergantikan di engineering modern: (1) Heaviside step H(t−a) untuk input mendadak / sinyal yang menyala-mati, (2) Dirac delta δ(t−a) untuk impulse / hammer test, (3) transfer function H(s) = Y(s)/U(s) sebagai representasi sistem, (4) konvolusi untuk respons ke input arbitrary, dan (5) sistem PD coupled untuk sistem multi-output.

Lima Konsep Aplikasi Laplace yang Wajib Dikuasai
H(t−a) ⇌ e−as/s ·   δ(t−a) ⇌ e−as ·   H(s) = Y(s)/U(s) ·   L{f*g} = F(s)·G(s) ·   sistem coupled di domain s
📈

Heaviside Step H(t−a)

Fungsi step Heaviside adalah representasi matematis dari sinyal mendadak (saklar dinyalakan, beban tiba-tiba, voltase step). Berbentuk 0 untuk t < a, 1 untuk t ≥ a. Transformasi: L{H(t−a)} = e−as/s. Faktor e−as = delay di domain s.

L{H(t−a)} = e^(−as)/s
⚡

Dirac Delta δ(t−a)

Fungsi impulse Dirac = "spike" instantan di t = a dengan luas 1. Model untuk hammer test, lightning strike, momentum impact. Transformasi: L{δ(t−a)} = e−as. Inilah impulse response, building block untuk semua respons sistem linier.

L{δ(t−a)} = e^(−as)
🎯

Transfer Function H(s)

Transfer function = rasio output ke input di domain s saat IVP nol: H(s) = Y(s)/U(s). Sifat sistem terkapsul dalam H(s): pole = stabilitas, zero = kemampuan filtering, gain = amplifikasi. Inilah representasi sistem yang dipakai engineer kendali.

H(s) = Y(s)/U(s)
∫

Konvolusi y = h*u

Teorema konvolusi: L{f*g} = F(s)·G(s) — perkalian di domain s = konvolusi di domain t. Konsekuensi: untuk sistem dengan transfer function H(s), y(t) = h(t)*u(t) = ∫₀^t h(τ)·u(t−τ)·dτ. Respons ke input arbitrary = konvolusi impulse response dengan input.

L{f*g} = F(s)·G(s)
Mengapa konsep ini penting di engineering? (1) Heaviside & Dirac adalah representasi matematis input nyata yang tidak smooth — voltase saklar, gaya hammer test, beban tiba-tiba. UC/VoP gagal total untuk input ini, hanya Laplace yang bisa. (2) Transfer function adalah jantung teori kendali: dari H(s), engineer dapat mendesain feedback controller, menghitung stabilitas (Routh-Hurwitz), dan analisis frekuensi (Bode plot). (3) Konvolusi memungkinkan menghitung output untuk input arbitrary — bila h(t) (impulse response) diketahui dari karakterisasi sistem, output untuk input apapun dapat dihitung tanpa solve PD ulang. (4) Sistem PD coupled (multi-output) muncul di robotika, jaringan sirkuit, sistem mekanik multi-DOF.
💡 Strategi belajar Pertemuan 11: (1) Pahami visualisasi — gambar Heaviside step (tangga di t=a), Dirac delta (panah panjang dengan area 1). (2) Hafal pasangan tabel baru — L{H(t−a)·f(t−a)} = e−as·F(s) (second shift). (3) Pikirkan sistem sebagai blok H(s) — input → block → output, semua di domain s. (4) Konvolusi visual — bayangkan sliding integral antara dua fungsi. (5) Latih dengan SymPy: sp.Heaviside(t-a), sp.DiracDelta(t-a), sp.laplace_transform().
Bagian 02

Heaviside Step H(t−a) —
Input Mendadak di Domain Laplace

Fungsi Heaviside step H(t−a) = representasi matematis dari sinyal yang mendadak menyala di t = a: 0 sebelum t = a, 1 setelahnya. Sangat penting untuk modeling input nyata: voltase saklar, gaya step load, kontrol on/off, sinyal digital. Transformasi Laplace: L{H(t−a)} = e−as/s. Faktor e−as = delay operator di domain s.

Definisi Heaviside dan Transformasi Laplace
H(t−a) = {0 jika t<a,   1 jika t≥a}  ⟹  L{H(t−a)} = e−as/s;    L{f(t−a)·H(t−a)} = e−as·F(s) (Second Shift)
📈

Definisi Step Function

Heaviside step H(t−a) = unit step function yang aktif di t = a. Untuk a = 0: H(t) = step di origin. Untuk a > 0: step ditunda a detik. Visual: tangga yang naik dari 0 ke 1 di t = a. Kemiringan tak-terhingga (step instan).

H(t−a) = step di t = a
🔁

Transformasi Heaviside

Dari definisi integral: L{H(t−a)} = ∫₀^∞ e−st·H(t−a) dt = ∫_a^∞ e−st dt = e−as/s. Kasus a = 0: L{H(t)} = 1/s = sama dengan L{1} (untuk t > 0, H(t) = 1).

L{H(t−a)} = e^(−as)/s
⏩

Second Shift Theorem

Teorema penting: L{f(t−a)·H(t−a)} = e−as·F(s). Fungsi yang aktif di t = a dengan bentuk f(t−a) (digeser) → faktor e−as·F(s) di domain s. Berguna untuk fungsi yang menyala dengan delay: cos(ω(t−a))·H(t−a), sin(ω(t−a))·H(t−a), polinomial dengan delay.

L{f(t−a)H(t−a)} = e^(−as)F(s)
🎯

Konstruksi Sinyal Kompleks

Heaviside step adalah building block: kombinasi step memberi rectangular pulse, gigi gergaji, square wave. Mis. pulse 0<t<5: p(t) = H(t) − H(t−5); transformasi: L{p(t)} = (1 − e−5s)/s. Pulse +H(t-10) − H(t-15): pulse kedua di t∈[10,15].

pulse = H(t) − H(t−d)
Cara menulis fungsi diskontinu dengan Heaviside:
(1) Step function tunda: f(t) = 5·H(t−2) = "0 sebelum t=2, lalu 5".
(2) Rectangular pulse: f(t) = 3·[H(t−1) − H(t−4)] = "0, lalu 3 dari t=1 sampai t=4, lalu 0".
(3) Sinyal yang aktif setelah delay dengan bentuk fungsi: f(t) = sin(2(t−3))·H(t−3) = sin(2t) tetapi mulai dari t=3.
(4) Saw-tooth via Heaviside: kombinasi step dan ramp.
Tiap term ditransformasi terpisah lalu dijumlahkan via linearitas.
Contoh 1 — fungsi step dengan delay: Hitung L{5·H(t−3)}.
Linearitas: L{5·H(t−3)} = 5·L{H(t−3)} = 5·e−3s/s.
Faktor 5 adalah magnitude step, e−3s adalah delay 3 detik di domain s.
Contoh 2 — pulse: Hitung L{2·[H(t) − H(t−4)]}.
= 2·(1/s − e−4s/s) = (2/s)·(1 − e−4s).
Pulse magnitudo 2 selama 4 detik.
Contoh 3 — second shift: Hitung L{(t−2)²·H(t−2)}.
Pakai second shift theorem: F(s) = L{t²} = 2/s³; L{(t−2)²·H(t−2)} = e−2s·2/s³.
⚠ Pitfall Heaviside: (1) Bedakan f(t)·H(t−a) (bentuk f tetap, tapi aktif setelah delay) dari f(t−a)·H(t−a) (bentuk f juga digeser). Hanya yang kedua match dengan second shift theorem. (2) Bila ada f(t)·H(t−a), tulis ulang sebagai f((t−a)+a)·H(t−a), lalu ekspansi. Mis. t·H(t−2) = ((t−2)+2)·H(t−2) = (t−2)·H(t−2) + 2·H(t−2). (3) SymPy sp.Heaviside(t-a) handle ini otomatis.
Bagian 03

Dirac Delta δ(t−a) —
Impulse & Hammer Test

Dirac delta function δ(t−a) = "spike" instantan di t = a dengan luas integral = 1. Bukan fungsi biasa (matematis: distribusi/generalized function), tetapi sangat penting untuk model impulse: hammer test struktur, lightning strike di sirkuit, momentum impact di sistem mekanik. Transformasi Laplace: L{δ(t−a)} = e−as. Saat a = 0: L{δ(t)} = 1 — paling sederhana.

Definisi Dirac Delta dan Sifat Sifting
δ(t−a):   ∫_{−∞}^∞ δ(t−a) dt = 1,   ∫_{−∞}^∞ f(t)·δ(t−a) dt = f(a) (sifting);    L{δ(t−a)} = e−as
⚡

Definisi Dirac Delta

δ(t−a) = limit dari fungsi narrow rectangular pulse: tinggi → ∞, lebar → 0, luas = 1. Visual: panah vertikal di t = a dengan label "1". Bukan fungsi biasa, tetapi distribusi Schwartz. Operasi: ∫f(t)·δ(t−a) dt = f(a) (sifting property — δ "menyaring" nilai f di t=a).

δ(t−a) = "spike" di t=a
🎯

Transformasi Laplace

Dari sifting: L{δ(t−a)} = ∫₀^∞ e−st·δ(t−a) dt = e−as untuk a ≥ 0. Untuk a = 0: L{δ(t)} = 1. Paling sederhana di antara semua transformasi! Inilah mengapa impulse response sangat fundamental untuk karakterisasi sistem.

L{δ(t−a)} = e^(−as)
🔨

Impulse Response h(t)

Untuk PD L(D)·y = δ(t) dengan IVP nol, solusi y(t) disebut impulse response h(t). Diperoleh via Laplace: L(D)·y = δ(t) → P(s)·H(s) = 1 → H(s) = 1/P(s), lalu invers Laplace. Inilah fingerprint sistem.

h(t) = L⁻¹{1/P(s)}
🔬

Aplikasi Engineering

Hammer test: pukul struktur dengan palu instrumented, ukur respons getarnya — itulah h(t). FFT dari h(t) memberi frequency response. Modal analysis: identifikasi frekuensi natural dari peak FFT. System identification: dari h(t) terukur, estimasi parameter sistem (m, c, k atau R, L, C).

FFT[h(t)] → frequency response
Sifat-sifat penting Dirac delta:
(1) Sifting: ∫f(t)·δ(t−a) dt = f(a) — δ mengambil nilai f di titik a.
(2) Symmetry: δ(−t) = δ(t) — fungsi genap.
(3) Scaling: δ(at) = δ(t)/|a| untuk a ≠ 0.
(4) Hubungan dengan Heaviside: δ(t) = dH(t)/dt — Dirac delta = turunan dari step function.
(5) Konvolusi dengan δ: f(t) * δ(t−a) = f(t−a) — konvolusi dengan delta = shift.
(6) Multiplikasi: f(t)·δ(t−a) = f(a)·δ(t−a) — sifting menjadi koefisien konstan.
Contoh 1 — solve PD dengan input impulse: Selesaikan y'' + 4y' + 13y = δ(t) dengan y(0) = 0, y'(0) = 0.
Step 1. Transformasi Laplace: (s² + 4s + 13)·Y(s) = 1 (karena L{δ(t)} = 1).
Step 2. Solve: Y(s) = 1/(s² + 4s + 13). Lengkapi kuadrat: 1/[(s+2)² + 9].
Step 3. Invers Laplace via tabel + first shift: L⁻¹{ω/[(s−a)² + ω²]} = eat·sin(ωt). Adjust faktor: 1/[(s+2)² + 9] = (1/3)·3/[(s+2)² + 9].
Solusi: h(t) = (1/3)·e−2t·sin(3t) = impulse response sistem ini. Pole −2 ± 3j → underdamped.
Contoh 2 — impulse di t = 2 detik: Selesaikan y'' + 9y = δ(t − 2) dengan IVP nol.
Step 1. Transformasi: (s² + 9)·Y = e−2s.
Step 2. Y(s) = e−2s/(s² + 9).
Step 3. Pakai second shift theorem: L⁻¹{e−as·F(s)} = f(t−a)·H(t−a). Di sini F(s) = 1/(s²+9) = (1/3)·3/(s²+9) → f(t) = (1/3)·sin(3t).
Solusi: y(t) = (1/3)·sin(3(t−2))·H(t−2) — sistem mulai berosilasi pada frekuensi natural 3 rad/s setelah impulse di t=2.
⚠ Tips Dirac delta: (1) δ(t) bukan fungsi biasa — jangan coba evaluasi di titik tertentu (nilainya "infinity" di t=a, "0" elsewhere). Yang punya makna adalah integral-nya. (2) SymPy sp.DiracDelta(t-a) handle ini secara simbolik. (3) Untuk numerik, aproksimasi δ dengan narrow rectangular pulse atau gaussian dengan σ → 0. (4) Impulse response unik untuk tiap sistem — dua sistem dengan h(t) sama adalah ekuivalen secara dinamis.
Bagian 04

Transfer Function H(s) —
Representasi Sistem di Domain Frekuensi

Transfer function adalah konsep terpenting dalam analisis sistem dinamis. Definisi: rasio Laplace output ke Laplace input pada syarat awal nol — H(s) = Y(s)/U(s). Ini bukan hanya formula matematis, melainkan representasi lengkap sifat sistem: pole menentukan stabilitas, zero menentukan filtering, gain menentukan amplifikasi. Engineer kendali, sirkuit, dan sinyal "berpikir" di domain s lewat transfer function.

Definisi Transfer Function dan Bentuk Standar
H(s) = Y(s)/U(s) (saat IVP nol);    H(s) = (bmsm + … + b₀) / (ansn + … + a₀);   pole = akar denominator,   zero = akar numerator
📐

Definisi H(s)

Untuk PD L(D)·y = u(t) dengan IVP nol, transformasi Laplace memberi L(s)·Y(s) = U(s), jadi H(s) = Y(s)/U(s) = 1/L(s). Lebih umum: H(s) = N(s)/D(s) rasio polinomial. Hanya bergantung pada parameter sistem, bukan pada input. Dua sistem dengan H(s) sama = ekuivalen secara dinamis.

H(s) = N(s)/D(s)
📍

Pole & Zero

Pole = akar denominator D(s). Menentukan karakter dinamik: real negatif → decay, real positif → unstable, kompleks → osilasi. Zero = akar numerator N(s). Menentukan frekuensi yang difilter (transmission zero). Lokasi pole/zero di s-plane mengungkap sifat sistem secara visual.

stabilitas: Re(pole) < 0
🎯

Output dari H(s)

Output untuk input apapun: Y(s) = H(s)·U(s), lalu invers Laplace. Mis. step input U(s) = 1/s → step response Y(s) = H(s)/s. Impulse U(s) = 1 → impulse response Y(s) = H(s) → h(t) = L⁻¹{H(s)}. Inilah mengapa h(t) ↔ H(s) pasangan yang sangat fundamental.

Y(s) = H(s)·U(s)
🌊

Frequency Response H(jω)

Substitusi s = jω di H(s) memberi frequency response H(jω). Magnitude |H(jω)| = gain pada frekuensi ω; phase ∠H(jω) = phase shift. Plot di Bode (log-log magnitude + linear phase) mengungkap perilaku filter (low-pass, high-pass, band-pass, notch). Inilah jembatan ke teori sinyal.

|H(jω)|, ∠H(jω) = Bode plot
Contoh — Transfer function sistem RLC seri: L·q'' + R·q' + (1/C)·q = u(t). Transformasi (IVP nol): (Ls² + Rs + 1/C)·Q(s) = U(s). Maka:
H(s) = Q(s)/U(s) = 1/(Ls² + Rs + 1/C).
Untuk L = 1 H, R = 4 Ω, 1/C = 13: H(s) = 1/(s² + 4s + 13) = 1/[(s+2)² + 9]. Pole: s = −2 ± 3j (underdamped). Tidak ada zero (numerator konstanta). Sistem ini stabil (Re(pole) = −2 < 0).
Klasifikasi sistem dari pole H(s):
(1) Pole real negatif: decay eksponensial (overdamped).
(2) Pole real berulang negatif: critically damped — fastest decay tanpa overshoot.
(3) Pole kompleks dengan Re < 0: underdamped — osilasi peluruh.
(4) Pole imajiner murni Re = 0: marginally stable — osilasi konstan (jarang stabil di praktik).
(5) Pole dengan Re > 0: unstable — output divergen (sistem tidak dapat dipakai).
Engineer kendali pakai feedback untuk menggeser pole ke half-plane kiri (stabilisasi).
⚠ Tips transfer function: (1) H(s) hanya valid bila IVP = 0. Bila IVP ≠ 0, ada term tambahan dari syarat awal — tidak murni transfer function lagi. (2) Untuk PD orde tinggi, faktorkan numerator dan denominator untuk identifikasi pole/zero. (3) Stabilitas: cek bahwa semua pole punya Re < 0. Ada satu pole di Re ≥ 0 → sistem unstable. (4) SymPy: H = sp.Function('H')(s) + sp.simplify(H_expr) untuk manipulasi simbolik.
Bagian 05

Konvolusi (Convolution) —
Output untuk Input Arbitrary

Teorema konvolusi adalah jewel kedua dari Laplace setelah transformasi turunan: L{f*g} = F(s)·G(s). Perkalian sederhana di domain s = konvolusi rumit di domain t. Konsekuensi luar biasa: untuk sistem dengan transfer function H(s), output untuk input apapun dapat dihitung sebagai y(t) = h(t)*u(t) = konvolusi impulse response dengan input. Tidak perlu solve PD ulang!

Definisi Konvolusi dan Teorema Konvolusi
(f*g)(t) = ∫₀^t f(τ)·g(t−τ)·dτ;    L{f*g} = F(s)·G(s);    y(t) = h(t)*u(t) (response sistem)
∫

Definisi Konvolusi

Konvolusi dua fungsi: (f*g)(t) = ∫₀^t f(τ)·g(t−τ)·dτ. Visual: f(τ) tetap, g(t−τ) di-flip dan slide di atas f, integral memberi area overlap. Sifat: komutatif (f*g = g*f), asosiatif, distributif ke penjumlahan. Identitas: f*δ = f (Dirac delta = identitas konvolusi).

(f*g)(t) = ∫₀^t f(τ)g(t−τ) dτ
⚡

Teorema Konvolusi

L{f*g} = F(s)·G(s). Bukti dari definisi: tukar urutan integrasi ganda. Kebalikan: L⁻¹{F(s)·G(s)} = f(t)*g(t) — bila Y(s) = perkalian dua F(s) yang invers individualnya diketahui, hasil di domain t = konvolusi mereka. Sangat berguna untuk invers Laplace bila pecahan parsial sulit.

L{f*g} = F(s)·G(s)
🎯

Output Sistem y = h*u

Untuk LTI system dengan transfer function H(s), input U(s) memberi Y(s) = H(s)·U(s). Invers Laplace via teorema konvolusi: y(t) = h(t)*u(t). Inilah fundamental theorem teori sistem linier: respons ke input arbitrary = konvolusi impulse response dengan input.

y(t) = h(t)*u(t)
🔬

Aplikasi Praktis

(1) Karakterisasi sistem via hammer test → dapat h(t). (2) Untuk input nyata u(t) (mis. earthquake record, audio signal), hitung response via konvolusi numerik scipy.signal.convolve(h, u). (3) Filter design: jendela waktu di domain t = filter di domain frekuensi. (4) Image processing: 2D convolution untuk blur, sharpen, edge detect.

np.convolve(h, u)
Contoh 1 — invers via konvolusi: Hitung L⁻¹{1/(s·(s+2))}.
Cara 1 (pecahan parsial): 1/(s(s+2)) = (1/2)/s − (1/2)/(s+2) → (1/2)·(1 − e−2t).
Cara 2 (konvolusi): Tulis sebagai produk: (1/s)·(1/(s+2)). L⁻¹{1/s} = 1; L⁻¹{1/(s+2)} = e−2t. Konvolusi: 1*e−2t = ∫₀^t 1·e−2(t−τ) dτ = e−2t·∫₀^t e2τ dτ = e−2t·[e2t−1]/2 = (1−e−2t)/2. ✓ Konsisten!
Contoh 2 — output sistem ke input arbitrary: Sistem dengan h(t) = e−t·H(t) menerima input ramp u(t) = t·H(t). Output:
y(t) = h(t)*u(t) = ∫₀^t e−τ·(t−τ)·dτ = (t−τ+1)·e−τ|₀^t = (1)·e−t − (t+1)·1·(−1) = e−t + t − 1.
Verifikasi via Laplace: H(s) = 1/(s+1), U(s) = 1/s², Y(s) = 1/[s²(s+1)]. Pecahan parsial: = 1/s² − 1/s + 1/(s+1) → y(t) = t − 1 + e−t. ✓ Cocok.
Konvolusi numerik dengan SciPy:
scipy.signal.convolve(h_array, u_array, mode='full') — untuk diskrit time series.
Misal: response struktur ke earthquake record: y = scipy.signal.convolve(h_struktur, u_earthquake) * dt.
Mode 'full' memberi konvolusi lengkap (panjang N+M−1); 'same' memberi panjang sama dengan input pertama.
Untuk continuous, gunakan scipy.integrate.quad dengan integrand h(τ)·u(t−τ).
⚠ Tips konvolusi: (1) Batas integrasi 0 sampai t (bukan −∞ sampai ∞) untuk causal systems. (2) Pakai konvolusi saat pecahan parsial sulit atau saat butuh insight tentang efek input historis ke output. (3) Konvolusi dengan delta function = shift: f(t)*δ(t−a) = f(t−a). (4) SymPy: tidak ada built-in konvolusi simbolik; pakai sp.integrate manual. (5) NumPy/SciPy untuk numerik; perhatikan dt scaling.
Bagian 06

PD Orde Tinggi —
Solve Orde 3, 4, n via Laplace

Salah satu keunggulan terbesar Laplace dibanding UC/VoP: PD orde tinggi diselesaikan dengan workflow yang sama seperti orde 2. Untuk PD orde n, transformasi turunan menghasilkan polinomial dalam s pangkat n, plus n syarat awal. Solve aljabar memberi Y(s), pecahan parsial dekomposisi denominator orde n menjadi term sederhana. UC/VoP gagal di sini karena ansatz menjadi terlalu rumit; Laplace tetap sistematik.

Transformasi Turunan Orde n untuk PD
L{f(n)(t)} = snF(s) − sn−1f(0) − sn−2f'(0) − … − f(n−1)(0);   n syarat awal: f(0), f'(0), …, f(n−1)(0)
3️⃣

PD Orde 3

Untuk PD y''' + ay'' + by' + cy = f(t), transformasi: L{y'''} = s³Y − s²y(0) − sy'(0) − y''(0). Butuh 3 syarat awal: y(0), y'(0), y''(0). Setelah aljabar: (s³+as²+bs+c)·Y(s) = … — denominator polinomial pangkat 3.

3 syarat awal untuk orde 3
4️⃣

PD Orde 4 (Defleksi Balok)

PD EI·y'''' = w(x) untuk defleksi balok = orde 4. Transformasi: L{y''''} = s⁴Y − s³y(0) − s²y'(0) − sy''(0) − y'''(0). Butuh 4 syarat batas (boundary conditions): defleksi, slope, momen, gaya geser. Sangat penting di structural engineering.

EI·y'''' = w(x)
🎯

Faktorisasi Denominator

Denominator polinomial pangkat n dapat difaktorkan via akar polinomial (numpy roots, SymPy solve). Tiap akar = pole sistem. Untuk akar real berbeda → pecahan parsial sederhana. Untuk akar kompleks (pasangan konjugat) → biarkan quadratic, lengkapi kuadrat.

D(s) = ∏ (s − r_i)
⚙️

Pecahan Parsial Orde Tinggi

Untuk denominator orde 3 atau lebih, pecahan parsial bisa rumit secara manual. Pakai SymPy sp.apart() atau NumPy scipy.signal.residue untuk dekomposisi otomatis. Hasil: jumlah term sederhana yang masing-masing dapat di-invers via tabel.

sp.apart(F, s) → ∑
Contoh — PD orde 3: Selesaikan y''' − 6y'' + 11y' − 6y = 0 dengan y(0) = 2, y'(0) = 5, y''(0) = 13 via Laplace.
Step 1. Transformasi: L{y'''} = s³Y − s²·2 − s·5 − 13 = s³Y − 2s² − 5s − 13; L{y''} = s²Y − 2s − 5; L{y'} = sY − 2; L{y} = Y.
Step 2. Substitusi: (s³ − 6s² + 11s − 6)·Y = 2s² + 5s + 13 − 6(2s + 5) + 11·2 = 2s² + 5s + 13 − 12s − 30 + 22 = 2s² − 7s + 5.
Step 3. Faktorkan denominator: s³ − 6s² + 11s − 6 = (s−1)(s−2)(s−3). Maka Y(s) = (2s² − 7s + 5)/[(s−1)(s−2)(s−3)].
Step 4. Pecahan parsial cover-up: A=(2−7+5)/((−1)(−2))=0; B=(8−14+5)/((1)(−1))=1; C=(18−21+5)/((2)(1))=1.
Hasil: Y(s) = 0/(s−1) + 1/(s−2) + 1/(s−3) → y(t) = e2t + e3t. Verifikasi: y(0)=2, y'(0)=2+3=5, y''(0)=4+9=13. ✓
Workflow umum PD orde n:
Langkah 1. Transformasi semua turunan via rumus L{f(k)}.
Langkah 2. Susun bentuk D(s)·Y(s) = N(s) dengan D = polinomial karakteristik pangkat n, N = numerator dari syarat awal + L{f(t)}.
Langkah 3. Faktorkan D(s) — cari semua akar (numpy.roots untuk numerik).
Langkah 4. Pecahan parsial — pakai sp.apart untuk orde tinggi.
Langkah 5. Invers Laplace tiap term via tabel.
Total: 5 langkah, sama untuk orde 2, 3, 4, atau lebih. Skalabilitas Laplace = standar untuk semua orde.
⚠ Tips PD orde tinggi: (1) Pastikan jumlah syarat awal = orde PD. (2) Untuk denominator orde 4+, faktorisasi mungkin tidak rapi (akar irrasional). Pakai numerik. (3) Bila ada pole kompleks, biarkan quadratic dan lengkapi kuadrat untuk match dengan e^(at)·sin/cos. (4) Untuk balok orde 4, syarat batas (BVP) berbeda dengan IVP — Laplace lebih natural untuk IVP. Untuk BVP, hasil Y(s) memuat parameter unknown yang harus dideterminasi dari syarat batas. (5) SymPy sp.dsolve dengan ics adalah cara cepat untuk PD orde tinggi.
Shortcut Formula untuk PD Linear Koefisien Konstan: Untuk PD orde-2 berbentuk a·y″ + b·y′ + c·y = r(t) dengan IVP y(0), y′(0), transformasi Laplace menghasilkan bentuk umum yang dapat ditulis langsung tanpa menderivasi setiap suku.
Shortcut Formula Y(s) — Orde 2 dan Orde 3
Bentuk umum      Y(s) = R(s)/Q(s) + G(s)/Q(s)
R(s)              = L{r(t)}              (transformasi sisi kanan)

Orde 2:    Q(s) = as² + bs + c  (polinomial karakteristik)
            G(s) = (as + b)·y(0) + a·y′(0)

Orde 3:    Q(s) = a₃s³ + a₂s² + a₁s + a₀
            G(s) = (a₃s² + a₂s + a₁)·y(0) + (a₃s + a₂)·y′(0) + a₃·y″(0)
Contoh 3.4 — Penerapan Shortcut pada PD Orde 3 Homogen: Selesaikan y‴ − 3y″ + 4y = 0 dengan syarat awal y(0) = 2, y′(0) = 3, y″(0) = 13.
Penyelesaian Langkah-demi-Langkah
Identifikasi    a₃ = 1, a₂ = −3, a₁ = 0, a₀ = 4     (homogen ⟹ R(s) = 0)
Q(s)          = s³ − 3s² + 4 = (s − 2)²·(s + 1)
G(s)          = (s² − 3s)·2 + (s − 3)·3 + 1·13
                = 2s² − 3s + 4
Substitusi     Y(s) = (2s² − 3s + 4) / [(s − 2)²·(s + 1)]
Pecahan parsial  Y(s) = 2/(s − 2)² + 1/(s + 1)
Solusi:      y(t) = 2t·e2t + e−t

Shortcut formula di atas dapat digeneralisasi untuk PD koefisien konstan orde-n apa pun: Q(s) selalu berupa polinomial karakteristik, sedangkan G(s) mengkompilasi seluruh kontribusi syarat awal dalam bentuk polinomial dengan koefisien yang merupakan kombinasi linier dari koefisien PD asli. Gunakan formula ini sebagai alat verifikasi cepat untuk hasil derivasi penuh IVP-Laplace yang dilakukan di atas.

Bagian 07

Sistem PD Coupled —
Multi-Output Lewat Laplace

Banyak sistem engineering memiliki multiple state yang saling terkait: arus dan tegangan di sirkuit multi-loop, posisi dan kecepatan di sistem mekanik multi-DOF, konsentrasi reaktan di reaktor kimia bertingkat. Dimodelkan sebagai sistem PD coupled — beberapa PD yang saling tergantung. Laplace mengubah sistem PD menjadi sistem aljabar linear yang dapat di-solve dengan metode matriks (Cramer, eliminasi Gauss).

Sistem PD Coupled di Domain Laplace
{x'(t) = a·x + b·y + f(t);   y'(t) = c·x + d·y + g(t)}  ⟹  matriks {[s−a, −b; −c, s−d]·[X(s); Y(s)] = [F(s)+x(0); G(s)+y(0)]}
🔗

Definisi Sistem Coupled

Dua atau lebih PD yang variabel-nya saling muncul di persamaan satu sama lain. Mis. x' = 2x − y, y' = x + 3y — x bergantung pada y, dan sebaliknya. Tidak bisa di-solve secara independen. Laplace ubah jadi sistem aljabar linear di domain s yang dapat dihitung via matriks.

x' = ax + by, y' = cx + dy
📊

Bentuk Matriks

Setelah transformasi: (sI − A)·X(s) = X(0) + F(s) dengan A matriks koefisien sistem, X(s) = [X₁(s), X₂(s), …]ᵀ vektor Laplace state, X(0) = vektor IVP, F(s) = vektor input Laplace. Matriks (sI − A) = generalisasi polinomial karakteristik untuk sistem.

(sI − A)X = X₀ + F
🧮

Solve via Matriks

Bila (sI − A) invertible (determinan ≠ 0): X(s) = (sI − A)⁻¹·(X(0) + F(s)). Untuk sistem 2×2, pakai rumus Cramer atau invers matriks 2×2 manual. Determinan det(sI − A) = polinomial karakteristik sistem — akar = eigenvalue matriks A.

X(s) = (sI − A)⁻¹·…
🎯

Eigenvalue & Stabilitas

Akar dari det(sI − A) = 0 = eigenvalue λ_i matriks A. Identik dengan akar karakteristik untuk sistem single PD. Sistem stabil bila semua eigenvalue Re(λ_i) < 0. Bila ada satu eigenvalue dengan Re ≥ 0 → unstable. Konektor langsung antara aljabar matriks (linear algebra) dengan teori sistem dinamis.

det(sI−A)=0 → eigenvalue
Contoh — sistem 2×2 sederhana: Selesaikan x' = 2x + y, y' = 3x + 4y dengan x(0) = 1, y(0) = 0.
Step 1. Transformasi: sX − 1 = 2X + Y → (s−2)X − Y = 1. Untuk y: sY − 0 = 3X + 4Y → −3X + (s−4)Y = 0.
Step 2. Sistem matriks: [s−2, −1; −3, s−4]·[X; Y] = [1; 0]. Determinan: (s−2)(s−4) − 3 = s² − 6s + 8 − 3 = s² − 6s + 5 = (s−1)(s−5).
Step 3. Cramer: X(s) = det([1,−1; 0,s−4])/det = (s−4)/[(s−1)(s−5)]. Y(s) = det([s−2,1; −3,0])/det = 3/[(s−1)(s−5)].
Step 4. Pecahan parsial X(s): cover-up: A=(1−4)/(1−5)=3/4; B=(5−4)/(5−1)=1/4. x(t) = (3/4)et + (1/4)e5t.
Untuk Y(s): A=3/(1−5)=−3/4; B=3/(5−1)=3/4. y(t) = −(3/4)et + (3/4)e5t.
Verifikasi: x(0) = 3/4 + 1/4 = 1 ✓; y(0) = −3/4 + 3/4 = 0 ✓.
Aplikasi nyata sistem coupled:
(1) Sirkuit multi-loop: Hukum Kirchhoff memberi sistem PD dengan arus i₁(t), i₂(t) di tiap loop.
(2) Sistem mekanik multi-DOF: Beberapa massa terhubung pegas — posisi tiap massa = state, sistem PD coupled.
(3) State-space representation: ẋ = Ax + Bu, y = Cx + Du — bahasa universal teori kendali modern.
(4) Sistem chemical reactor: Konsentrasi reaktan di tiap CSTR (Continuous Stirred-Tank Reactor) saling terkait via flow.
(5) Population dynamics: Lotka-Volterra (predator-prey) saat dilinearisasi sekitar equilibrium.
⚠ Tips sistem PD coupled: (1) Jumlah PD = jumlah state (variabel tak diketahui). Untuk n state, butuh n IVP. (2) Pakai matriks notation untuk efisiensi — tertulis kompak dan langsung map ke linear algebra. (3) Untuk sistem n > 2, pakai NumPy np.linalg.solve atau SymPy sp.linsolve. (4) Bila eigenvalue kompleks, sistem dinamis menampilkan osilasi — sesuai akar karakteristik kompleks pada PD single. (5) Dekomposisi via eigenvektor (decoupling) memberi insight: sistem coupled dapat dipisah menjadi modes independen pada axis eigen.

Aplikasi Penerapan Laplace

Tools ini mengubah Laplace dari sekedar metode matematis menjadi standar industri di teori kendali, sirkuit, vibrasi, sinyal, dan thermal engineering modern.

🏗️ Analogi 1 — Modal Analysis dengan Hammer Test (Dirac δ) Engineer struktur melakukan hammer test untuk karakterisasi struktur bangunan/jembatan/airframe: pukul instrumented hammer ke struktur, ukur respons getar via accelerometer. Input ≈ Dirac impulse δ(t); respons = impulse response h(t). Lewat Laplace: H(s) = X(s)/F(s) = 1/(ms² + cs + k). FFT dari h(t) memberi frequency response H(jω) — peak di plot mengungkap frekuensi natural struktur. Dipakai untuk: validasi model FEM, deteksi damage (frekuensi natural berubah bila ada crack), identifikasi mode shape.
⚡ Analogi 2 — Desain Kontroler PID (Transfer Function) PID controller = jantung kendali industri (motor, suhu, level cairan). Transfer function PID: C(s) = K_p + K_i/s + K_d·s. Sistem closed-loop: T(s) = (G·C)/(1 + G·C) dengan G plant transfer function. Engineer tuning K_p, K_i, K_d untuk mencapai: (a) zero steady-state error (lewat term integral K_i/s), (b) response cepat (lewat K_p), (c) tanpa overshoot besar (lewat term derivative K_d·s). Stabilitas dicek via lokasi pole closed-loop di s-plane. Tanpa konsep transfer function dari Laplace, kendali otomatis tidak ada.
🌡️ Analogi 3 — Sirkuit RC dengan Pulse Train (Heaviside) Digital signal di sirkuit elektronik = sequence of pulses, dimodelkan dengan kombinasi Heaviside: u(t) = V₀·∑[H(t−nT) − H(t−nT−d)] = pulse berulang T detik dengan duty d. Transformasi Laplace: U(s) = (V₀/s)·(1−e−ds)·∑e−nTs = (V₀/s)·(1−e−ds)/(1−e−Ts) (deret geometri). Output sirkuit RC: Y(s) = U(s)/(τs+1). Invers Laplace memberi response time domain — penting untuk integrity sinyal di IC, analisis switching power supply, dan koreksi distorsi sinyal digital.
🎯 Analogi 4 — Earthquake Response via Konvolusi Engineer struktur menghitung respons bangunan terhadap earthquake ground motion u(t) (record dari accelerometer di lapangan). Bangunan dimodelkan sebagai sistem orde 2: mẍ + cẋ + kx = −m·ü_g(t). Karena u(t) input arbitrary (tidak smooth, bukan eksponensial/sin), konvolusi ideal: x(t) = h(t)*u(t) = ∫₀^t h(τ)·u(t−τ)·dτ dengan h(t) = impulse response struktur. Implementasi numerik: scipy.signal.convolve(h, u). Hasilnya = response history yang dipakai untuk: (a) cek apakah amplitudo melebihi threshold safety, (b) desain damper untuk mereduksi peak response, (c) evaluasi LRFD building code.
💰 Analogi 5 — Robot Lengan 2-Link (Sistem PD Coupled) Robot lengan 2-link planar punya 2 joint (θ₁, θ₂) dengan dinamik coupled (gerakan θ₁ memengaruhi torque di θ₂ lewat Coriolis). Linearisasi sekitar konfigurasi tertentu: sistem PD coupled M·θ̈ + C·θ̇ + K·θ = τ(t) dengan M, C, K matriks 2×2. Laplace: (Ms² + Cs + K)·Θ(s) = T(s). Solve via matrix invers: Θ(s) = (Ms² + Cs + K)⁻¹·T(s). Pole = eigenvalue dari sistem (dapatkan dari det = 0). Robot designer pakai ini untuk: (a) verifikasi stabilitas saat motor failure, (b) tuning gain feedback, (c) desain trajectory smoothing. Sistem coupled ada di mana-mana di robotika, kendaraan multi-DOF, dan jaringan listrik.

(3) Konvolusi memungkinkan response untuk input arbitrary dari karakterisasi h(t) saja. (4) Sistem coupled handle multi-state yang muncul di robotika, kendali multi-loop, kimia bertingkat. (5) PD orde tinggi diselesaikan sistematik. Penguasaan kelima konsep ini = passport ke teori kendali, DSP, control engineering, dan analisis dinamika sistem di tingkat profesional. import sympy as sp

t, s = sp.symbols('t s', positive=True)

# Contoh 1: L{H(t-a)} = e^(-as)/s
F1 = sp.laplace_transform(sp.Heaviside(t - 3), t, s, noconds=True)
print("L{H(t-3)} =", F1)   # exp(-3s)/s

# Contoh 2: Pulse rectangular: H(t) - H(t-2)
pulse = sp.Heaviside(t) - sp.Heaviside(t - 2)
F2 = sp.laplace_transform(pulse, t, s, noconds=True)
print("L{pulse} =", sp.simplify(F2))
# (1 - exp(-2s))/s

# Contoh 3: L{δ(t-a)} = e^(-as)
F3 = sp.laplace_transform(sp.DiracDelta(t - 2), t, s, noconds=True)
print("L{δ(t-2)} =", F3)   # exp(-2s)

# Contoh 4: Solve PD dengan input impulse: y'' + 4y' + 13y = δ(t)
y = sp.Function('y')
eq = sp.Eq(y(t).diff(t, 2) + 4*y(t).diff(t) + 13*y(t), sp.DiracDelta(t))
sol = sp.dsolve(eq, y(t), ics={y(0): 0, y(t).diff(t).subs(t, 0): 0})
print("Impulse response h(t) =", sp.simplify(sol.rhs))
# (1/3)*exp(-2t)*sin(3t)*Heaviside(t)

# Contoh 5: Second shift theorem - L{f(t-a)·H(t-a)} = e^(-as)·F(s)
# Verifikasi: L{(t-2)²·H(t-2)} = e^(-2s)·2/s³
F5 = sp.laplace_transform((t - 2)**2*sp.Heaviside(t - 2), t, s, noconds=True)
print("L{(t-2)²·H(t-2)} =", sp.simplify(F5))   # 2*exp(-2s)/s³

import sympy as sp
import numpy as np
import matplotlib.pyplot as plt

t, s = sp.symbols('t s', positive=True)
X, Y = sp.symbols('X Y')

# Sistem PD coupled:
#   x'(t) = 2x + y
#   y'(t) = 3x + 4y
# IVP: x(0) = 1, y(0) = 0

# Transformasi Laplace
# sX - 1 = 2X + Y  =>  (s-2)X - Y = 1
# sY - 0 = 3X + 4Y =>  -3X + (s-4)Y = 0

A = sp.Matrix([[s - 2, -1], [-3, s - 4]])
b = sp.Matrix([1, 0])

# Solve via matrix inverse
sol = A.inv() * b
X_s = sp.simplify(sol[0])
Y_s = sp.simplify(sol[1])
print("X(s) =", X_s)   # (s-4)/[(s-1)(s-5)]
print("Y(s) =", Y_s)   # 3/[(s-1)(s-5)]

# Eigenvalue (akar dari det(sI - A) = 0)
det = sp.simplify(A.det())
eigvals = sp.solve(det, s)
print("Eigenvalue:", eigvals)   # [1, 5]

# Invers Laplace
x_t = sp.inverse_laplace_transform(X_s, s, t)
y_t = sp.inverse_laplace_transform(Y_s, s, t)
print("x(t) =", sp.simplify(x_t))   # (3/4)e^t + (1/4)e^(5t)
print("y(t) =", sp.simplify(y_t))   # -(3/4)e^t + (3/4)e^(5t)

# Plot
t_num = np.linspace(0, 1, 100)
x_num = (3/4)*np.exp(t_num) + (1/4)*np.exp(5*t_num)

Penyelesaian Langkah-demi-Langkah
Q(s)            = s² − 3s + 2 = (s − 1)·(s − 2)
G(s)            = (s − 3)·4 + 5 = 4s − 7

Verifikasi dengan SymPy: Hasil manual di atas dapat dibandingkan langsung dengan keluaran sp.dsolve() maupun pendekatan shortcut Y(s) = R(s)/Q(s) + G(s)/Q(s) yang dilanjutkan sp.apart(). 
import sympy as sp
t, s = sp.symbols('t s', positive=True)
y = sp.Function('y')(t)

# Pendekatan 1: dsolve langsung
eq  = sp.Eq(y.diff(t, 2) - 3*y.diff(t) + 2*y, 0)
sol = sp.dsolve(eq, y, ics={y.subs(t, 0): 4, y.diff(t).subs(t, 0): 5})
print(sol)               # Eq(y(t), 3*exp(t) + exp(2*t))

# Pendekatan 2: shortcut Laplace Y(s) = R(s)/Q(s) + G(s)/Q(s)
R, Q, G = 0, s**2 - 3*s + 2, (s - 3)*4 + 5
Ys      = (R + G) / Q
print(sp.apart(Ys, s))   # 3/(s - 1) + 1/(s - 2)

Modal Analysis Hammer Test

Engineer struktur menggunakan hammer test untuk karakterisasi mesin/struktur: input impulse Dirac δ(t), output impulse response h(t) = transfer function di domain waktu. Gunakan tools Pertemuan 11 — Dirac delta, transfer function H(s), dan analisis pole — untuk validasi modal sistem dan identifikasi frekuensi natural.

Skenario

Modal Analysis Mesin via Hammer Test:
Identifikasi Sistem dengan Dirac & Transfer Function

📋 KASUS KARAKTERISASI STRUKTUR

Engineer maintenance pabrik melakukan hammer test (impulse test) pada bracket mesin centrifugal yang dicurigai bermasalah. Bracket dipukul dengan instrumented hammer (input ≈ impulse Dirac), respons getarnya diukur dengan accelerometer di titik yang sama. Hasil pengukuran h(t) menunjukkan osilasi peluruh: h(t) ≈ A·e−σt·sin(ωd·t) dengan σ = 2 (decay rate), ωd = 3 rad/s (damped frequency), A ≈ 0.33. Tim diminta pakai Transformasi Laplace untuk: (a) cari transfer function H(s), (b) identifikasi pole sistem, (c) verifikasi dengan dsolve PD asal, dan (d) bandingkan dengan FFT untuk frequency response.

Input: F(t) = F₀·δ(t) (hammer)
L{δ(t)} = 1 → Y(s) = H(s)
h(t) = (1/3)·e−2t·sin(3t)
pole = −2 ± 3j (underdamped)

Bracket sebagai sistem mass-spring-damper berisi PD orde 2 m·ẍ + c·ẋ + k·x = F(t). Hammer test = experimental modal analysis: input impulse Dirac membangkitkan semua mode getar, respons h(t) berisi semua frekuensi natural sebagai exponentially decaying sinusoid. Tiga pertanyaan diskusi: (1) hubungan h(t) dengan transfer function H(s), (2) identifikasi parameter sistem dari pole, (3) interpretasi engineering — apakah sistem stabil, dan implikasi maintenance.

— Visualisasi impulse response h(t) dari hammer test: osilasi pada frekuensi natural ωd = 3 rad/s dengan envelope decay e−σt (σ = 2). Pole kompleks s = −2±3j memberi karakter underdamped — sistem stabil tetapi berosilasi sebelum settling.

Petunjuk Penyelesaian — Akar Kompleks via Completing the Square: Kasus hammer test ini merupakan aplikasi langsung dari pola PD orde-2 dengan akar karakteristik kompleks. Sebagai pemanasan, perhatikan PD referensi y″ + 6y′ + 13y = 0 dengan y(0) = 2, y′(0) = 1 yang diselesaikan via completing the square pada denominator.
Pola Solusi PD Orde-2 dengan Akar Kompleks Konjugat
Polinomial karakteristik   s² + 6s + 13 = (s + 3)² + 4
Akar kompleks           s = −3 ± 2j       (σ = −3, ωd = 2)
Bentuk umum solusi       y(t) = eσt·(A·cos ωdt + B·sin ωdt)
Substitusi IVP            A = 2,   B = 7/2
Solusi:              y(t) = 2·e−3t·cos 2t + (7/2)·e−3t·sin 2t
Aplikasi pada Hammer Test — Sudut Pandang Terbalik (System Identification): Pada kasus forum ini, alur penyelesaian dibalik. Mulai dari h(t) yang terukur secara eksperimental, lakukan: (i) tebak Q(s) dengan completing the square dari pola decay-osilasi, (ii) identifikasi pole sistem dari akar kompleks s = σ ± ωdj, (iii) rekonstruksi PD m·ẍ + c·ẋ + k·x = F(t) untuk memperoleh parameter fisik (massa, damping, kekakuan). Inilah esensi system identification — kebalikan dari arah forward (PD → solusi) menjadi arah inverse (solusi → PD).

01

Buktikan bahwa output h(t) = L⁻¹{H(s)} untuk input impulse Dirac δ(t). Lalu temukan transfer function H(s) dari h(t) terukur.

›

Untuk PD m·ẍ + c·ẋ + k·x = F(t) dengan IVP nol, transfer function: H(s) = X(s)/F(s) = 1/(ms² + cs + k). Saat input F(t) = δ(t): L{δ(t)} = 1, jadi X(s) = H(s)·1 = H(s). Invers Laplace: h(t) = L⁻¹{H(s)} = impulse response. Inilah mengapa h(t) langsung mengkode H(s) sistem. Dari h(t) terukur (1/3)·e−2t·sin(3t), identifikasi: pole kompleks s = −2 ± 3j → denominator (s+2)² + 9 = s² + 4s + 13; numerator (1/3)·3 = 1 (sesuai L{(1/3)e^(-2t)sin(3t)} = (1/3)·3/[(s+2)²+9]). Maka H(s) = 1/(s² + 4s + 13).

02

Identifikasi parameter sistem (m, c, k) dari pole H(s) = 1/(s² + 4s + 13). Asumsikan m = 1 kg. Hitung c, k, dan damping ratio ζ.

›

PD: m·ẍ + c·ẋ + k·x = F(t) → transfer function H(s) = 1/(ms² + cs + k). Bandingkan dengan 1/(s² + 4s + 13): koefisien s² adalah m, koefisien s adalah c, koefisien konstan adalah k. Dengan m = 1: c = 4 N·s/m, k = 13 N/m. Frekuensi natural ωn = √(k/m) = √13 ≈ 3.61 rad/s. Damping ratio ζ = c/(2√(mk)) = 4/(2√13) = 2/√13 ≈ 0.555. Dibandingkan dengan damped frequency: ωd = ωn·√(1−ζ²) = 3.61·√(1−0.308) = 3.61·0.832 = 3.0 rad/s ✓ cocok dengan pole imajiner!

💬 Hubungan pole ↔ parameter: untuk pole s = −σ ± jωd: σ = c/(2m) = 4/2 = 2 ✓; ωn² = k/m → ωn = √13; ζ = σ/ωn = 2/3.61 ≈ 0.55 (underdamped, 0 < ζ < 1). Sistem akan settling time ≈ 4/σ = 2 detik, dengan overshoot ≈ exp(−πζ/√(1−ζ²)) ≈ 13%. Inilah system identification dari hammer test: dari output (h(t) terukur) ke parameter sistem (m, c, k).
QUICK CHECK — Apa makna damping ratio ζ ≈ 0.55?
Sistem overdamped (tidak ada osilasi)
Underdamped (0 < ζ < 1) — ada osilasi peluruh, response cepat dengan moderate overshoot
Critically damped (response tercepat tanpa overshoot)
Unstable (pole di kanan)
✅ Benar! ζ ≈ 0.55 adalah underdamped (0 < ζ < 1). Karakter: (a) osilasi peluruh — pole kompleks dengan envelope decay e−σt. (b) Settling time ≈ 4/σ = 2 s (5% rule). (c) Overshoot moderat ≈ 13% — tidak terlalu besar (yang akan terjadi bila ζ→0) atau lambat (yang akan terjadi bila ζ→1+). Untuk power supply atau control system, ζ ∈ [0.6, 0.8] sering dipilih sebagai kompromis ideal: response cepat tanpa overshoot berlebihan. Dari hammer test, ζ < 0.1 menunjukkan struktur "ringing" (vibrasi terlalu lama) — perlu damper.
❌ Damping ratio ζ klasifikasi: ζ = 0 (undamped, osilasi tak meluruh), 0 < ζ < 1 (underdamped, osilasi peluruh), ζ = 1 (critically damped, no osilasi), ζ > 1 (overdamped, slow decay). Untuk pole −2 ± 3j: σ = 2, ωd = 3, ωn = √(σ²+ωd²) = √13 ≈ 3.61, ζ = σ/ωn ≈ 0.55 → underdamped.
