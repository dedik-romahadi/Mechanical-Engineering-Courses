Bagian 01

Apa Itu
Getaran Mesin?

Getaran mesin adalah gerakan osilasi (bolak-balik) komponen mesin di sekitar posisi keseimbangan. Fenomena ini terjadi karena ketidakseimbangan rotasi, misalignment, bearing aus, atau gaya eksternal periodik. Getaran adalah sinyal kondisi mesin — terlalu besar berarti ada masalah, terlalu kecil bisa berarti mesin mati.

Persamaan Gerak Harmonik Sederhana
x(t) = A sin(ωt + φ)
📐

Gerakan Harmonik Sederhana

Model paling dasar getaran. Posisi massa berubah secara sinusoidal terhadap waktu dengan amplitudo konstan jika tidak ada redaman. Ini adalah building block untuk semua analisis getaran lebih kompleks.

x(t) = A sin(ωt + φ)
🔄

Frekuensi Alami (fₙ)

Frekuensi getaran bebas sistem — frekuensi di mana sistem akan bergetar sendiri jika diberi gangguan awal lalu dilepas. Ditentukan oleh rasio kekakuan terhadap massa.

fₙ = (1/2π)·√(k/m)
⚡

Frekuensi Sudut (ωₙ)

Laju perubahan sudut fase per satuan waktu (rad/s). Versi "matematis" dari frekuensi linier — lebih nyaman untuk persamaan diferensial. Hubungan: \(\omega = 2\pi f\).

ωₙ = 2π·fₙ = √(k/m)
📊

Amplitudo (A)

Simpangan maksimum dari posisi keseimbangan. Menentukan besarnya energi getaran. Satuan: mm atau m. Amplitudo terlalu besar → kerusakan komponen, fatigue failure pada mesin.

A = x_maks = |x_min|
⏱️

Periode & Frekuensi

Periode \(T\) adalah waktu untuk satu siklus penuh (detik). Frekuensi \(f\) adalah banyaknya siklus per detik (Hz). Keduanya saling kebalikan: tinggi frekuensi = pendek periode.

T = 1/f = 2π/ω
🎯

Sudut Fase (φ)

Menentukan posisi awal osilasi pada waktu \(t = 0\). Dua getaran dengan \(\varphi\) berbeda dikatakan punya beda fase (phase shift). Penting saat membandingkan sinyal getaran di titik berbeda pada mesin.

φ = arctan(ẋ₀ / ωx₀)
⚠️ Resonansi — Fenomena Paling Berbahaya:
Ketika frekuensi gaya eksitasi mendekati frekuensi alami sistem (\(\omega \to \omega_n\)), amplitudo getaran melonjak drastis. Pada sistem tanpa redaman, amplitudo teoritis → ∞. Inilah mengapa analisis frekuensi alami sangat kritis dalam desain mesin — kita harus memastikan frekuensi operasi mesin jauh dari frekuensi alami strukturnya.
💡 Mengapa Pelajari Getaran? Setiap mesin yang berputar atau bergerak akan bergetar. Getaran berlebihan menyebabkan kerusakan bearing, fatigue retak, kebisingan, dan ketidaknyamanan operator. Sebaliknya, analisis getaran adalah salah satu metode predictive maintenance paling powerful — sinyal getaran membawa "tanda tangan" kondisi mesin yang bisa dideteksi sebelum kerusakan terjadi.
Bagian 02

Parameter Getaran
yang Diukur

Tiga besaran fundamental yang diukur dalam analisis getaran mesin: perpindahan (displacement), kecepatan (velocity), dan percepatan (acceleration). Ketiganya saling terkait melalui turunan terhadap waktu, dan masing-masing punya kegunaan dominan di rentang frekuensi berbeda.

Hubungan Antar Parameter (Gerakan Harmonik)
x(t) = A sin(ωt)  →  ẋ(t) = Aω cos(ωt)  →  ẍ(t) = −Aω² sin(ωt)
📏

Perpindahan (Displacement)

Posisi instan benda dari titik keseimbangan. Satuan: mm atau µm. Paling informatif untuk getaran frekuensi rendah (di bawah 10 Hz). Sensor: proximity probe (eddy current).

x(t) [mm]
💨

Kecepatan (Velocity)

Turunan pertama perpindahan. Satuan: mm/s. Indikator utama kerusakan bearing dan gearbox, paling efektif untuk frekuensi 10–1000 Hz. Standar industri: ISO 10816 (level kelas mesin). Sensor: velocity transducer.

v_max = Aω [mm/s]
⚡

Percepatan (Acceleration)

Turunan kedua perpindahan, atau gaya per massa (\(F = ma\)). Satuan: m/s² atau g. Paling sensitif untuk getaran frekuensi tinggi (di atas 1000 Hz) seperti bearing defect, gear mesh. Sensor: accelerometer (piezoelectric).

a_max = Aω² [m/s²]
📈

RMS vs Peak vs Peak-to-Peak

Tiga cara mengukur "besar" getaran. Peak = nilai maksimum, Peak-to-Peak = jarak puncak ke lembah (2A untuk sinusoidal), RMS = root-mean-square (\(A/\sqrt{2}\) untuk sinusoidal). Industri umumnya pakai RMS untuk overall vibration level.

RMS = A/√2 ≈ 0.707·A
📐 Kapan Gunakan Parameter Mana?
Pilih parameter yang menghasilkan magnitude paling konstan di rentang frekuensi yang ingin diamati. Frekuensi rendah → displacement dominan, frekuensi menengah → velocity, frekuensi tinggi → acceleration. Menggunakan parameter yang salah bisa menyembunyikan signal penting di noise floor.
Bagian 03

Klasifikasi
Jenis Getaran

Getaran diklasifikasikan berdasarkan dua dimensi: ada-tidaknya gaya eksitasi terus-menerus (bebas vs paksa), dan ada-tidaknya elemen redaman yang menyerap energi (teredam vs tidak teredam). Empat kombinasi ini membentuk dasar seluruh analisis getaran mekanik.

〰️

Getaran Bebas (Free Vibration)

Terjadi setelah gangguan awal (impulse, simpangan awal, kecepatan awal), tanpa gaya luar yang berkelanjutan. Sistem bergetar pada frekuensi alaminya (\(\omega_n\)). Contoh: garpu tala dipukul, jembatan setelah truk lewat.

mẍ + cẋ + kx = 0
⚡

Getaran Paksa (Forced Vibration)

Ada gaya eksitasi periodik berkelanjutan \(F(t)\). Frekuensi respons mengikuti frekuensi eksitasi (\(\omega\)), bukan frekuensi alami. Contoh: motor unbalance saat berputar, mesin reciprocating, getaran akibat angin pada cerobong asap.

mẍ + cẋ + kx = F₀sin(ωt)
🚫

Tanpa Redaman (Undamped)

Sistem ideal tanpa elemen disipasi energi (\(c = 0\)). Amplitudo konstan selamanya untuk getaran bebas. Tidak realistis di mesin nyata, tapi penting untuk analisis frekuensi alami dan resonansi (di mana redaman = 0 → amplitudo \(\to \infty\)).

mẍ + kx = 0
🔽

Teredam (Damped)

Ada elemen peredam (dashpot, friction, internal damping) yang mendisipasi energi. Amplitudo berkurang secara eksponensial untuk getaran bebas. Rasio redaman \(\zeta\) menentukan perilaku: under-, critically-, atau overdamped.

x(t) = Ae^(−ζωₙt)·sin(ω_d·t)
Tiga Mode Redaman (berdasarkan ζ)
ζ < 1 — Underdamped  ·  ζ = 1 — Critically damped  ·  ζ > 1 — Overdamped
🎯 Catatan untuk Desainer Mesin:
Mayoritas mesin nyata adalah sistem underdamped (\(\zeta\) sekitar 0.01–0.1) — amplitudo getaran bebas berkurang lambat sehingga osilasi tetap terlihat selama beberapa siklus. Pemilihan redaman optimal adalah trade-off antara: cukup tinggi untuk meredam resonansi, tapi tidak terlalu tinggi sehingga membuat sistem kaku dan mengurangi efisiensi mekanik.
Bagian 04

Persamaan Gerak
Getaran Bebas SDOF

Sistem Single Degree of Freedom (SDOF) adalah model paling sederhana yang menangkap esensi getaran. Satu massa terhubung ke dinding via pegas (kekakuan \(k\)) dan damper (koefisien \(c\)). Persamaan diferensial yang menggambarkan geraknya adalah fondasi dari seluruh analisis getaran.

Persamaan Gerak — SDOF Bebas Teredam
mẍ + cẋ + kx = 0
⚖️

Tiga Gaya yang Berimbang

Persamaan adalah keseimbangan: gaya inersia (\(m\ddot{x}\), Newton 2) + gaya redaman (\(c\dot{x}\), proporsional kecepatan) + gaya pegas (\(kx\), proporsional simpangan) = 0. Tanpa gaya luar, semua energi berpindah antara energi kinetik massa, energi potensial pegas, dan disipasi damper.

F_inersia + F_damper + F_pegas = 0
🔢

Frekuensi Natural ωₙ

Dari kasus tanpa redaman (\(c=0\)): \(m\ddot{x} + kx = 0\). Solusi sinusoidal memberi \(\omega_n = \sqrt{k/m}\). Ini adalah fingerprint sistem — hanya bergantung pada massa dan kekakuan, tidak pada amplitudo atau kondisi awal.

ωₙ = √(k/m)
📉

Rasio Redaman ζ

Bilangan tak berdimensi yang menormalkan redaman: \(\zeta = c / (2\sqrt{km})\). Membandingkan damping aktual dengan damping kritis. Nilai khas: kayu ~0.01, beton ~0.02, baja ~0.005, karet ~0.05–0.1.

ζ = c / (2√(km))
🌀

Frekuensi Teredam ω_d

Untuk sistem underdamped (\(\zeta < 1\)), osilasi terjadi pada frekuensi sedikit lebih rendah dari \(\omega_n\). Untuk damping kecil (\(\zeta < 0.1\)), \(\omega_d \approx \omega_n\) — perbedaan dapat diabaikan dalam praktik.

ω_d = ωₙ·√(1−ζ²)
Solusi Underdamped (ζ < 1) — Osilasi Meluruh
x(t) = A·e^(−ζωₙt)·sin(ω_d·t + φ)
💡 Logaritmik Decrement δ — Cara Mengukur ζ dari Eksperimen: Jika kita amati getaran bebas dan ukur amplitudo dua puncak berurutan \(x_1\) dan \(x_2\), maka \(\delta = \ln(x_1/x_2) = 2\pi\zeta/\sqrt{1-\zeta^2}\). Ini cara praktis menentukan rasio redaman sistem nyata tanpa perlu tahu nilai \(c\) langsung.
Bagian 05

Analogi Getaran
dalam Kehidupan Sehari-hari

Konsep getaran, frekuensi alami, dan redaman akan lebih mudah dipahami melalui pengalaman nyata yang sering kita temui setiap hari. Lima analogi di bawah ini — dari ayunan taman sampai jam mekanik — semuanya mengikuti persamaan getaran yang sama, hanya dengan parameter \(m, c, k\) yang berbeda.

🎢

Ayunan Anak di Taman

Tarik ayunan ke samping lalu lepas — ayunan akan berosilasi pada frekuensi alaminya, lalu perlahan berhenti karena gesekan udara (damping ringan). Periode ayunan hanya bergantung pada panjang tali dan gravitasi, bukan amplitudo (untuk sudut kecil).

T = 2π√(L/g)
🚗

Suspensi Mobil

Saat melewati polisi tidur, mobil terangkat lalu turun — tetapi tidak terus berosilasi berkat shock absorber (damper). Sistem mobil dirancang sebagai underdamped (\(\zeta \approx 0.3\)): satu osilasi kecil lalu balik stabil. Terlalu banyak damping = kasar, terlalu sedikit = "boat-feeling".

ζ ≈ 0.3 (optimal)
🥁

Gendang/Drum yang Dipukul

Membran drum mengalami gangguan awal (pukulan), lalu berosilasi pada frekuensi alaminya menghasilkan suara. Drum besar = membran lebih berat = frekuensi lebih rendah = nada bass. Drum kecil = frekuensi tinggi = nada treble. Pure free vibration dengan damping akustik.

f ∝ √(tegangan/massa)
⏰

Jam Mekanik Pendulum

Pendulum berosilasi pada frekuensi alami yang sangat presisi (\(T = 2\pi\sqrt{L/g}\)) — itulah sebabnya digunakan untuk mengukur waktu. Sedikit dorongan dari mekanisme escapement mengganti energi yang hilang karena gesekan, menjaga osilasi tetap berlangsung selamanya.

T = 2π√(L/g)
🌉

Jembatan Tacoma Narrows (1940)

Contoh klasik resonansi destruktif: angin dengan frekuensi tertentu memicu mode getaran torsional jembatan. Karena redaman struktur sangat rendah, amplitudo membesar terus hingga jembatan runtuh. Pelajaran: dalam desain struktur, harus dianalisis SEMUA mode getaran natural untuk dihindari frekuensi eksitasi yang mungkin terjadi.

ω_eksitasi → ωₙ ⇒ runtuh
🎸

Senar Gitar yang Dipetik

Petik senar = beri gangguan awal → senar berosilasi pada frekuensi alaminya. Frekuensi tergantung tegangan, panjang, dan massa per panjang senar. Bunyi perlahan menghilang karena disipasi energi ke udara dan body gitar. Pemain bass pakai senar tebal (massa tinggi) → frekuensi rendah.

f = (1/2L)·√(T/μ)
💡 Pola yang Sama di Semua Analogi: Setiap kasus di atas mengikuti struktur persamaan yang sama: \(m\ddot{x} + c\dot{x} + kx = F(t)\) — hanya parameter \(m, c, k\) dan kondisi gaya \(F\) yang berbeda. Kekuatan matematika getaran adalah universalitasnya: satu persamaan diferensial mendeskripsikan ayunan, jembatan, mobil, gitar, dan jam pendulum sekaligus.

Animasi Interaktif
Konsep Getaran

Eksplorasi konsep-konsep kunci yang sudah dipelajari secara visual dan real-time. Geser slider untuk mengubah parameter dan amati perubahan instan pada grafik. Animasi di bawah memvisualisasikan rasio redaman, respons waktu, dan fenomena resonansi — konsep fundamental yang akan terus digunakan di pertemuan-pertemuan berikutnya.

Animasi 1 — Kurva DMF dan Sudut Fase vs Rasio Frekuensi r
── ζ₁ DMF (cyan) ── ζ₂ DMF (amber) ── ζ₃ DMF (pink) ┄ Fase φ(ζ₂) — skala kanan 0–180°
📊 Cara Membaca Grafik Ini: Sumbu X = \(r\) (rasio frekuensi \(\omega/\omega_n\), dari 0 hingga 2.5). Sumbu Y kiri = DMF (amplifikasi amplitudo). Garis merah putus-putus di \(r=1\) menandai resonansi. Garis hijau di \(r=\sqrt{2}\) menandai batas isolasi transmissibility.
Amati: (1) Semakin kecil \(\zeta\), puncak DMF semakin tinggi dan tajam. (2) Semua kurva bertemu di DMF=1 saat \(r=0\) (statis). (3) Saat \(r \gg 1\), semua kurva mendekati 0 — massa tidak bergerak mengikuti gaya frekuensi tinggi. (4) Garis putus-putus (fase \(\varphi\)) naik dari 0° ke 180° — laju kenaikan paling cepat di dekat resonansi.
Animasi 2 — Gaya Eksitasi vs Respons Steady-State dalam Domain Waktu
── Gaya eksitasi F(t) = F₀sin(ωt) ── Respons steady-state x(t) = X sin(ωt−φ)
📊 Cara Membaca Grafik Ini: Kurva amber = gaya eksitasi \(F(t)\) (normalisasi); kurva cyan = respons perpindahan \(x(t)\) steady-state. Perhatikan pergeseran fase: respons tertinggal \(\varphi\) dari eksitasi.
Amati: (1) Saat \(r \to 0\): \(\varphi \approx 0°\) — dua kurva hampir sefase, amplitudo respons ≈ \(X_{st}\). (2) Saat \(r = 1\): \(\varphi = 90°\) — respons tertinggal seperempat siklus, amplitudo besar. (3) Saat \(r \gg 1\): \(\varphi \to 180°\) — respons berlawanan fase, amplitudo kecil.
Animasi 3 — Build-Up Respons (Transien + Steady-State) dengan RK4
── Respons total x(t) = transien + steady-state -- Amplop steady-state ±X
📊 Cara Membaca Grafik Ini: Dihitung dengan metode numerik Runge-Kutta orde 4 (RK4) — metode yang sama yang digunakan di Jupyter Notebook (lihat Bagian 08 Cell 3). Sumbu X = waktu (0 hingga 30 s). Amplop merah putus-putus = nilai steady-state teoritis \(\pm X\).
Amati: (1) Di awal (\(t\) kecil), respons berosilasi tidak beraturan — ini fase transien. (2) Secara bertahap, amplitudo "menyesuaikan diri" ke nilai steady-state (amplop merah). (3) Semakin kecil \(\zeta\), semakin lama transien berlangsung sebelum mencapai steady-state. (4) Saat \(r \approx 1\) dan \(\zeta\) kecil, awal respons bisa jauh melampaui amplop steady-state — inilah kondisi berbahaya saat mesin dinyalakan.
Animasi 4 — Kurva Transmissibility (Tᵣ) vs r: Efektivitas Isolasi Getaran
📊 Cara Membaca Grafik Ini: Sumbu Y = Transmissibility \(T_r\) (rasio getaran yang diteruskan ke fondasi). Garis putus-putus horizontal = \(T_r = 1\) (batas). Garis hijau vertikal = \(r = \sqrt{2} \approx 1.414\).
Interpretasi: (1) Area kiri \(r=\sqrt{2}\) — \(T_r > 1\), semua isolator justru MEMPERBESAR getaran ke fondasi. (2) Area kanan \(r=\sqrt{2}\) — \(T_r < 1\), isolasi efektif, getaran berkurang sebelum mencapai fondasi. (3) Di area isolasi (\(r>\sqrt{2}\)): kurva dengan \(\zeta\) LEBIH KECIL memberikan isolasi LEBIH BAIK. Ini berlawanan dengan intuisi — terlalu banyak redaman justru buruk untuk isolasi di frekuensi tinggi. (4) Kesimpulan desain: gunakan isolator lunak (\(k\) kecil → \(\omega_n\) kecil → \(r\) besar) dengan redaman cukup (\(\zeta \approx 0.1–0.15\)) untuk mesin berfrekuensi tinggi.
Bagian 07

Aplikasi Getaran
di Industri

Analisis getaran bukan sekadar latihan teori — ini salah satu tools paling powerful di dunia industri untuk menjaga mesin tetap sehat, menghemat biaya downtime, dan mencegah kecelakaan. Di bawah ini beberapa aplikasi nyata yang wajib diketahui insinyur mesin.

🔍

Predictive Maintenance

Monitoring getaran rutin (bulanan, mingguan, atau real-time) untuk mendeteksi kerusakan mesin sebelum terjadi. Setiap jenis fault (unbalance, misalignment, bearing defect) punya tanda tangan frekuensi khas yang muncul di spektrum getaran. Hemat hingga 30–40% biaya maintenance dibanding reactive maintenance.

ISO 10816 · ISO 13373
⚖️

Balancing Rotor

Rotor yang tidak seimbang menghasilkan gaya sentrifugal periodik → getaran pada frekuensi \(1 \times RPM\). Balancing (static + dynamic) meminimalkan amplitudo getaran. Kritis untuk turbin, impeller pompa, kipas industri, dan rotor motor listrik. Tanpa balancing yang baik, bearing aus 3–5× lebih cepat.

G-grade (ISO 1940)
🎯

Machine Alignment

Misalignment poros (motor–pompa, motor–gearbox) menghasilkan getaran pada \(2 \times RPM\) di arah aksial. Alignment yang buruk → stress tambahan di bearing, kopling aus, seal bocor. Teknik modern: laser alignment dengan toleransi 0.05 mm/mm.

Angular + Offset error
🚨

Kontrol Getaran Struktur

Gedung tinggi, jembatan, dan platform offshore dirancang dengan tuned mass damper atau isolator seismik untuk meredam getaran akibat angin, gempa, atau beban dinamis. Contoh: Taipei 101 menggunakan TMD 660 ton di puncak untuk meredam getaran angin.

f_TMD ≈ f_struktur
🏥

Diagnostik Medis

Stetoskop adalah vibration sensor untuk deteksi kelainan jantung dan paru. Ultrasonografi memanfaatkan gelombang mekanik frekuensi tinggi. Terapi: ultrasonic untuk fisioterapi, USG untuk scan jaringan — semua aplikasi getaran terkontrol di bidang medis.

2–18 MHz (USG)
⚠️

Vibration Safety Standard

Standar ISO 5349 membatasi paparan getaran di tangan (hand-arm vibration), ISO 2631 untuk getaran seluruh tubuh (whole-body vibration). Eksposur berlebihan → sindrom Raynaud, nyeri punggung, hilang fungsi sendi. Mesin dengan getaran tinggi wajib dilengkapi isolator atau sistem damping.

Cell 1 — Setup Parameter & Hitung ωₙ, fₙ, ζ, ω_d Python 
import numpy as np

# ═══ Parameter Sistem Referensi (Pertemuan 1 — Getaran Bebas SDOF) ═══
m  = 5       # massa (kg)
k  = 2000    # kekakuan pegas (N/m)
c  = 40      # koefisien redaman (Ns/m)

# ═══ Perhitungan Parameter Getaran ═══
omega_n = np.sqrt(k / m)                 # frekuensi natural (rad/s)
f_n     = omega_n / (2 * np.pi)             # frekuensi natural (Hz)
T_n     = 1 / f_n                         # periode natural (s)
c_c     = 2 * np.sqrt(k * m)             # redaman kritis (Ns/m)
zeta    = c / c_c                          # rasio redaman (dimensionless)
omega_d = omega_n * np.sqrt(1 - zeta**2)  # frek. teredam (rad/s)
f_d     = omega_d / (2 * np.pi)             # frek. teredam (Hz)

print(f"ωₙ  = {omega_n:.4f} rad/s")
print(f"fₙ  = {f_n:.4f} Hz")
print(f"T   = {T_n:.4f} s")
print(f"ζ   = {zeta:.4f}  ({'Underdamped' if zeta<1 else 'Critically damped' if zeta==1 else 'Overdamped'})")
print(f"ω_d = {omega_d:.4f} rad/s  (deviasi dari ωₙ: {abs(omega_n-omega_d)/omega_n*100:.2f}%)")
Cell 2 — Solusi Analitik Underdamped (Bandingkan 3 Mode ζ) Python 
import matplotlib.pyplot as plt
import numpy as np

# Kondisi awal: simpangan 0.1 m, kecepatan 0
x0, v0 = 0.1, 0
m, k = 5, 2000
omega_n = np.sqrt(k / m)

def free_vibration(t, zeta):
    """Solusi analitik getaran bebas SDOF"""
    if zeta < 1:   # Underdamped
        omega_d = omega_n * np.sqrt(1 - zeta**2)
        A = np.sqrt(x0**2 + ((v0 + zeta*omega_n*x0) / omega_d)**2)
        phi = np.arctan2(x0*omega_d, v0 + zeta*omega_n*x0)
        return A * np.exp(-zeta*omega_n*t) * np.sin(omega_d*t + phi)
    elif zeta == 1:  # Critically damped
        return (x0 + (v0 + omega_n*x0)*t) * np.exp(-omega_n*t)
    else:             # Overdamped
        s1 = (-zeta + np.sqrt(zeta**2-1)) * omega_n
        s2 = (-zeta - np.sqrt(zeta**2-1)) * omega_n
        C1 = (v0 - s2*x0) / (s1 - s2)
        C2 = x0 - C1
        return C1*np.exp(s1*t) + C2*np.exp(s2*t)

t = np.linspace(0, 3, 1000)
fig, ax = plt.subplots(figsize=(10, 5))

for zeta, col, lbl in [(0.05, '#0ea5e9', 'Underdamped (ζ=0.05)'),
                    (1.0,  '#f97316', 'Critically damped (ζ=1)'),
                    (2.0,  '#ef4444', 'Overdamped (ζ=2)')]:
    ax.plot(t, free_vibration(t, zeta), color=col, lw=2, label=lbl)

ax.axhline(y=0, color='white', lw=0.5, alpha=0.3)
ax.set_xlabel('Waktu t (s)'); ax.set_ylabel('Perpindahan x (m)')
ax.set_title('Tiga Mode Redaman — Respons Getaran Bebas')
ax.legend(); ax.grid(alpha=0.2)
plt.tight_layout(); plt.show()
Cell 3 — Simulasi Numerik RK4: Getaran Bebas Underdamped Python 
import numpy as np
import matplotlib.pyplot as plt

m, k, c = 5, 2000, 40
omega_n = np.sqrt(k/m)
zeta    = c / (2*np.sqrt(k*m))

def free_vib(t, y):
    """RK4 state: y = [x, x_dot]. PD: mẍ + cẋ + kx = 0"""
    x, xd = y
    xdd = (-c*xd - k*x) / m
    return np.array([xd, xdd])

dt = 0.001
t  = np.arange(0, 5, dt)
y  = np.array([0.1, 0.0])  # kondisi awal: x=0.1 m, xdot=0
X_all = []

for ti in t:
    X_all.append(y[0])
    # Runge-Kutta Orde 4
    k1 = free_vib(ti, y)
    k2 = free_vib(ti+dt/2, y+dt/2*k1)
    k3 = free_vib(ti+dt/2, y+dt/2*k2)
    k4 = free_vib(ti+dt, y+dt*k3)
    y = y + dt/6 * (k1 + 2*k2 + 2*k3 + k4)

X_all = np.array(X_all)

# Envelope eksponensial: x(t) maksimum = x0 * exp(-zeta*omega_n*t)
envelope = 0.1 * np.exp(-zeta*omega_n*t)

plt.figure(figsize=(10, 4))
plt.plot(t, X_all, color='#00e09e', lw=1.2, label='x(t) RK4')
plt.plot(t,  envelope, color='#ef4444', ls='--', alpha=0.7, label=f'±{0.1}·exp(−ζωₙt) envelope')
plt.plot(t, -envelope, color='#ef4444', ls='--', alpha=0.7)
plt.xlabel('Waktu t (s)'); plt.ylabel('Perpindahan x (m)')
plt.title(f'Getaran Bebas Underdamped — ζ={zeta:.3f}, ωₙ={omega_n:.2f} rad/s')
plt.legend(); plt.grid(alpha=0.2); plt.tight_layout(); plt.show()
Cell 4 — Logarithmic Decrement: Ekstrak ζ dari Sinyal Eksperimen Python 
import numpy as np
from scipy.signal import find_peaks
import matplotlib.pyplot as plt

# Simulasi data "eksperimen" getaran bebas — bayangkan ini dari akselerometer
m, k, c = 5, 2000, 40
omega_n_true = np.sqrt(k/m)
zeta_true    = c / (2*np.sqrt(k*m))
omega_d_true = omega_n_true * np.sqrt(1-zeta_true**2)

t = np.linspace(0, 4, 2000)
x = 0.1 * np.exp(-zeta_true*omega_n_true*t) * np.cos(omega_d_true*t)

# ═══ Ekstraksi ζ dari puncak-puncak sinyal ═══
peaks, _ = find_peaks(x, height=0)
x_peaks  = x[peaks]
t_peaks  = t[peaks]

# Logarithmic decrement antara puncak ke-1 dan ke-n
n = 5   # selisih siklus
delta = (1/n) * np.log(x_peaks[0] / x_peaks[n])
zeta_est = delta / np.sqrt(4*np.pi**2 + delta**2)

# Plot hasil
fig, ax = plt.subplots(figsize=(10, 4))
ax.plot(t, x, color='#0ea5e9', lw=1.2, label='Signal x(t)')
ax.plot(t_peaks, x_peaks, 'ro', label='Puncak-puncak terdeteksi')
ax.set_xlabel('Waktu t (s)'); ax.set_ylabel('x (m)')
ax.set_title('Logarithmic Decrement — Identifikasi ζ dari Data')
ax.grid(alpha=0.2); ax.legend()
plt.tight_layout(); plt.show()

print(f"δ (log decrement): {delta:.4f}")
print(f"ζ estimasi:        {zeta_est:.4f}")
print(f"ζ true value:      {zeta_true:.4f}")
print(f"Error:             {abs(zeta_est-zeta_true)/zeta_true*100:.2f}%")

Diagnosa Awal —
Pompa Sentrifugal Bermasalah

📋 KASUS INDUSTRI

Sebuah pompa sentrifugal di pabrik kimia mulai menunjukkan gejala bermasalah: getaran terasa di lantai, kebisingan meningkat, dan operator melaporkan baut penopang motor longgar berulang kali dalam beberapa minggu terakhir. Tim maintenance baru saja memasang accelerometer untuk monitoring rutin, dan data awal menunjukkan amplitudo getaran melebihi batas ISO 10816 untuk kelas mesin ini.

Sebagai mahasiswa baru di mata kuliah Getaran Mekanik, Anda diminta melakukan diagnosa awal berbasis konsep dasar yang baru dipelajari — sebelum tim senior melakukan analisis spektrum FFT lebih mendalam di pertemuan berikutnya.

RPM motor: 1500
v_RMS: 8.2 mm/s
Massa pompa: 120 kg
Frekuensi dominan: 25 Hz

Pengukuran v_RMS = 8.2 mm/s berarti getaran termasuk kategori "unsatisfactory" (D zone) menurut ISO 10816 untuk pompa berukuran ini. Frekuensi dominan dari analisis sederhana adalah 25 Hz, sangat dekat dengan frekuensi putar motor (1500 RPM = 25 Hz). Forum ini mengajak Anda mendiagnosa apakah ini gejala unbalance, misalignment, atau resonansi, dan mengusulkan langkah investigasi awal.

Apa diagnosa awal yang paling mungkin untuk pompa ini?

›

Konversi 1500 RPM ke Hz, lalu bandingkan dengan frekuensi dominan terukur (25 Hz). Berdasarkan tabel diagnostik di Bagian 07, jelaskan kemungkinan penyebab utama. Diskusikan juga apa beda antara unbalance, misalignment, dan resonansi — ketiganya bisa memberi gejala mirip tetapi penyebab fisik berbeda.

f = RPM/60 1× RPM → unbalance 2× RPM → misalignment
QUICK CHECK — Frekuensi dominan 25 Hz pada motor 1500 RPM termasuk frekuensi karakteristik yang mana?
1× RPM (25 Hz) — gejala klasik unbalance rotor
2× RPM (50 Hz) — gejala misalignment poros
BPFO (bearing outer race) — defect bearing
Sub-sinkron (< 1× RPM) — oil whirl di journal bearing
✅ Tepat! 1500 RPM ÷ 60 = 25 Hz, dan frekuensi dominan terukur tepat 25 Hz = 1× RPM. Ini gejala klasik unbalance rotor — distribusi massa rotor tidak simetris terhadap sumbu putar. Solusi: dynamic balancing sesuai standar ISO 1940.
❌ Konversi dulu: 1500 RPM = 1500/60 = 25 Hz. Frekuensi dominan terukur (25 Hz) sama persis dengan 1× RPM. Lihat tabel diagnostik di Bagian 07 untuk identifikasi fault.

Mengapa parameter v_RMS dipilih sebagai indikator utama, bukan perpindahan atau percepatan?

›

Berdasarkan Bagian 02 (Parameter Getaran), tiap parameter dominan di rentang frekuensi yang berbeda. Standar ISO 10816 secara spesifik menggunakan v_RMS untuk monitoring mesin rotasi. Diskusikan: kenapa kecepatan (mm/s), bukan perpindahan (mm) atau percepatan (m/s²)?

💬 Pertimbangkan: Frekuensi operasi mesin biasa 10–1000 Hz. Pada rentang ini, hubungan antar parameter v = A·ω, a = A·ω² berarti perpindahan menjadi terlalu kecil di frekuensi tinggi (susah diukur), sedangkan percepatan menjadi terlalu besar di frekuensi tinggi (didominasi noise). Velocity = "sweet spot" yang relatif konstan.
QUICK CHECK — Untuk pompa pada 25 Hz, parameter mana yang paling tepat untuk monitoring rutin?
Perpindahan (mm) — paling akurat di frekuensi rendah saja (< 10 Hz)
Kecepatan (mm/s) — paling efektif di rentang 10–1000 Hz, sesuai ISO 10816
Percepatan (m/s²) — paling baik untuk frekuensi tinggi (> 1000 Hz, bearing defect)
RMS dari ketiganya — kompleks dan tidak ada standar industri
✅ Benar! Pada 25 Hz (rentang 10–1000 Hz), velocity adalah parameter paling informatif dan relatif konstan terhadap frekuensi. Itulah sebabnya ISO 10816 (standar dunia untuk vibration severity mesin rotasi) berbasis v_RMS dengan threshold sesuai kelas mesin. v_RMS = 8.2 mm/s pada pompa kelas ini = zona "unsatisfactory" → wajib investigasi.
❌ Lihat Bagian 02 untuk pemilihan parameter berdasarkan rentang frekuensi. Velocity paling sesuai untuk frekuensi mesin rotasi tipikal (10–1000 Hz). Standar ISO 10816 secara eksplisit berbasis v_RMS.

  4. Verifikasi — buka terminal baru lalu ketik: conda --version

conda --version

conda create -n getaran_mekanik python=3.11 -y

conda activate getaran_mekanik

Test Setup — Hello Vibration!

Buat file baru test_setup.ipynb di VS Code, pastikan kernel Python (Getaran Mekanik) dipilih, lalu paste & run cell di bawah:

test_setup.ipynb — Cell 1
import numpy as np
import matplotlib.pyplot as plt

# Sistem getaran bebas: m·ẍ + k·x = 0  →  ωₙ = √(k/m)
m, k = 1.0, 100.0
omega_n = np.sqrt(k/m)

t = np.linspace(0, 2, 500)
x = 0.1 * np.cos(omega_n * t)

plt.figure(figsize=(8, 3))
plt.plot(t, x, color='#10b981', linewidth=2)
plt.title(f'Hello Vibration! ωₙ = {omega_n:.2f} rad/s')
plt.xlabel('Waktu (s)'); plt.ylabel('x(t) [m]')
plt.grid(alpha=0.3); plt.tight_layout(); plt.show()
