Bagian 01

Konsep Modal Analysis:
Dari 2 DoF ke MDoF Umum

Pertemuan 7 sudah membahas frekuensi pribadi dan mode shape untuk sistem 2 DoF. Pertanyaan berikutnya: bagaimana untuk sistem dengan banyak derajat kebebasan (MDoF) — gedung 10 lantai, jembatan rangka, mesin pesawat? Modal Analysis adalah generalisasi yang sangat powerful: setiap MDoF system dapat didekomposisi menjadi N mode independen, masing-masing seperti sistem 1-DoF sederhana. Inilah jembatan teori 1 DoF → real-world structures.

Insight Kunci
Sistem N-DoF memiliki N frekuensi pribadi dan N mode shape. Setiap mode "decoupled" — bisa dianalisis secara terpisah. Solusi total = superposisi kontribusi tiap mode.
🏢

MDoF di Dunia Nyata

Gedung 10 lantai = 10 DoF translasi (per lantai). Jembatan suspension = 100+ DoF. Sayap pesawat (FEM) = 10⁴-10⁶ DoF. Setiap struktur kompleks intrinsik adalah MDoF — analisis 1-DoF/2-DoF hanyalah simplifikasi pedagogis.

Real structure: N = 10 - 10⁶ DoF
🎯

Idea Sentral: Decoupling

Persamaan gerak MDoF asli kompleks: [M]{ẍ} + [K]{x} = {F} — N persamaan saling terkait (coupled). Modal analysis transforms ke N persamaan independen (decoupled) — masing-masing seperti 1-DoF sederhana. Setelah decouple, solve trivial.

Coupled MDoF → N independent 1-DoF
🌊

Tiga Tahap Modal Analysis

(1) Solve eigenvalue problem: dapatkan N frekuensi pribadi ωi & N mode shape φi. (2) Modal transformation: ubah koordinat fisik {x} ke koordinat modal {η} via [Φ]. (3) Solve N independent equations in η, lalu transform balik.

Eigen → Transform → Solve → Inverse
🎼

Analogi: Spektrum Suara

Mode shape MDoF system seperti not-not di partitur. Setiap "not" punya frekuensi (mode i) dan amplitudo (kontribusi ηi). Total respons = chord — kombinasi simultan semua not. Modal analysis = mengurai chord ke not penyusunnya.

x(t) = Σ φ_i · η_i(t) (modal sum)
Mengapa Modal Analysis Powerful?
Untuk sistem 100-DoF (mis. gedung 100 lantai), persamaan gerak [M]{ẍ}+[K]{x}={F} adalah 100 ODE coupled — solving direct sangat lambat. Setelah modal transformation, jadi 100 ODE independent — solve paralel, atau hanya kontribusi beberapa mode dominan (biasanya mode 1-5 cukup untuk 95% akurasi). Reduksi komputasi 10-100×. Inilah mengapa modal analysis adalah workhorse di structural dynamics, FEM software (ANSYS, Abaqus, Nastran), seismic engineering.
💡 Konvensi Notasi: Mahasiswa harus paham notasi standar. {x} atau x = vektor displacement N-dim (koordinat fisik). {η} = vektor modal (koordinat normal mode). [M], [K] = matriks N×N (mass & stiffness). φi = mode shape ke-i (vektor N-dim). [Φ] = modal matrix (matriks N×N berisi semua mode shape sebagai kolom).
Bagian 02

Generalized Eigenvalue Problem:
[K] − ω²[M] = 0

Untuk sistem MDoF tak teredam, solusi harmonik {x(t)} = {X}·sin(ωt) di-substitusi ke [M]{ẍ} + [K]{x} = 0 menghasilkan generalized eigenvalue problem: ([K] − ω²[M]){X} = 0. Solusi non-trivial: det([K] − ω²[M]) = 0 — polinomial karakteristik berderajat N dengan N akar (frekuensi pribadi). Untuk MDoF, solving manual rumit; pakai numerik.

Generalized Eigenvalue Problem
[K]{φ} = ω²[M]{φ}  ⟺  det([K] − ω²[M]) = 0
📐

Versi Standar vs Generalized

Standard eigenvalue: A·v = λ·v. Generalized eigenvalue: A·v = λ·B·v (struktural dinamika selalu dengan dua matriks M dan K). Numerical libraries: scipy.linalg.eigh(K, M) atau numpy.linalg.eig(inv(M)@K).

A·v = λ·B·v (generalized)
🔢

N Eigenvalues = N Frekuensi Pribadi

Polinomial det([K]−ω²[M])=0 berderajat N → N akar untuk ω². Akar selalu positif (untuk K positive definite, system stable) → N nilai ωi positif. Konvensi: ordered ω1 < ω2 < ... < ωN.

N values: ω₁ < ω₂ < ... < ω_N
🎯

Setiap ωi Punya Mode Shape φi

Untuk tiap eigenvalue ωi², solve ([K] − ωi²[M])·{φi} = 0 untuk vektor non-trivial φi (mode shape). Mode shape adalah vektor N-dim — menjelaskan pola gerak relatif tiap massa di mode i.

([K] − ω_i²[M]){φ_i} = 0
⚡

Komputasi Modern

Untuk MDoF besar (N > 100), pakai iterative algorithms: Lanczos, Arnoldi, subspace iteration. Library: scipy.sparse.linalg.eigsh(), MATLAB eigs(). Hanya hitung beberapa eigenvalue terendah (lowest modes paling penting untuk dinamika).

For large N: only solve k < N modes
Mengapa Frekuensi Rendah Lebih Penting?
Untuk sistem N-DoF, mode dengan ω rendah punya energi tertinggi dan paling mudah ter-excited oleh load realistic (gempa, angin, putaran mesin). Mode dengan ω tinggi (mis. ω50 untuk N=100) hampir tidak pernah ter-excited di praktik. Aturan praktis: solve 5-10 lowest modes saja sudah cukup untuk 95% akurasi response. Hemat komputasi orders of magnitude.
💡 Numerical Stability: Direct inversion inv(M) @ K bisa numerically unstable untuk M yang ill-conditioned. Lebih baik pakai scipy.linalg.eigh(K, M) yang langsung solve generalized eigenvalue dengan algoritma stable (Cholesky decomposition + standard eigenvalue). Hampir selalu pilih route ini di production code.
Bagian 03

Mode Shape & Normalization:
RASIO, Bukan Absolute Amplitude

Mode shape φi menjelaskan pola relatif gerak tiap massa di mode i — bukan amplitude absolut. Saat dapat eigenvector dari numerical solver, magnitude-nya arbitrary (tergantung implementasi library). Untuk konsistensi & analisis lanjut, mode shape biasanya di-normalize dengan satu dari tiga konvensi standar.

Tiga Konvensi Normalisasi
Mass-normalized · Max-normalized (φmax = 1) · Unit-norm (||φ|| = 1)
⚖️

Mass-Normalized: φᵀMφ = 1

Setiap mode shape di-scale agar φiᵀ[M]φi = 1. Konsekuensi: φiᵀ[K]φi = ωi². Konvensi paling powerful untuk modal analysis lanjut — modal mass = 1, modal stiffness = ω². Standard di FEM software.

φ_i^T M φ_i = 1, φ_i^T K φ_i = ω_i²
📏

Max-Normalized: φmax = 1

Element terbesar di mode shape = 1 (atau −1). Mudah interpretasi visual: mode shape directly readable sebagai "rasio relative ke titik dengan amplitudo terbesar". Default umum di textbook untuk pedagogis.

max(|φ_i|) = 1
🎯

Unit-Norm: ||φ|| = 1

Euclidean norm = 1 (sama seperti unit vector). Default output dari numpy.linalg.eig(). Tidak punya physical meaning langsung, tapi mathematically convenient untuk operasi linear algebra.

||φ_i|| = √(φ_i^T φ_i) = 1
🔄

Konversi Antar Konvensi

Mudah convert: hitung scaling factor untuk current mode, divide dengan target normalization. Mass-normalized: φnorm = φ / √(φᵀMφ). Max-norm: φnorm = φ / max(|φ|). Engineer harus konsisten dalam satu analisis.

φ_normalized = φ / scaling_factor
Mass-Normalization adalah Standar Industri
Mengapa mass-normalized paling banyak dipakai? Karena response equations menjadi simple:
• Modal mass: mi = φiᵀMφi = 1 (no scaling needed)
• Modal stiffness: ki = φiᵀKφi = ωi²
• Modal damping (Rayleigh): ci = φiᵀCφi = 2ζiωi
• Modal force: fi(t) = φiᵀ{F(t)}
Modal equation untuk tiap mode: η̈i + 2ζiωiη̇i + ωi²ηi = fi(t) — bentuk standard 1-DoF! Solve N independent equations, super clean.
💡 Visualisasi Mode Shape: Plot mode shape sebagai bar chart atau garis menghubungkan posisi tiap massa. Mode 1 biasanya monotonic (semua sama tanda). Mode 2 punya 1 zero crossing. Mode 3 punya 2 zero crossings. Mode N−1 zero crossings. Pattern ini membantu identifikasi mode dari data eksperimen.
Bagian 04

Orthogonality Conditions:
Mengapa Mode-Mode Bisa "Decoupled"

Property paling fundamental dari mode shape: orthogonal terhadap matriks M dan K. Untuk dua mode berbeda i ≠ j: φiᵀMφj = 0 dan φiᵀKφj = 0. Inilah kunci matematis yang memungkinkan modal decoupling. Tanpa orthogonality, modal analysis tidak akan bekerja.

Orthogonality Conditions
φiᵀ[M]φj = 0 (i≠j)  ·  φiᵀ[K]φj = 0 (i≠j)
🔗

Orthogonal terhadap M (Mass Orthogonality)

φiᵀ[M]φj = 0 untuk i ≠ j. Bukti via eigenvalue equation: kalikan satu equation dengan φjᵀ, transposed equation dengan φi, subtract. Karena ωi ≠ ωj, terbukti orthogonal.

M-orthogonal: φ_i^T M φ_j = 0
🔧

Orthogonal terhadap K (Stiffness Orthogonality)

Sama analoginya: φiᵀ[K]φj = 0 untuk i ≠ j. Konsekuensi langsung dari M-orthogonality dan eigenvalue equation. Kedua orthogonality ini bersamaan disebut "M-K orthogonality".

K-orthogonal: φ_i^T K φ_j = 0
🌐

Bukan Geometric Orthogonality

Hati-hati: mode shape tidak orthogonal dalam sense biasa (φiᵀφj = 0). Hanya orthogonal terhadap M dan K. Ini disebut "M-orthogonality" atau "weighted orthogonality" — generalization dari orthogonality biasa.

φ_i^T φ_j ≠ 0 generally (NOT geometric ortho)
⚡

Konsekuensi untuk Decoupling

Substitusi {x} = [Φ]{η} ke equation of motion, lalu pre-multiply dengan [Φ]ᵀ. Hasilnya: off-diagonal terms semua nol (karena orthogonality). Persamaan jadi diagonal = N independent equations.

Φ^T M Φ = diag, Φ^T K Φ = diag
Visual Interpretation Orthogonality
Orthogonality M-orthogonal physically berarti: tiap mode tidak "merasakan" mass distribusi mode lain. Mode 1 dengan pola tertentu memberikan momentum yang tidak bertukar dengan mode 2 (asal mass-orthogonal). Inilah yang memungkinkan decoupling — mode-mode independent secara dinamis. Konsekuensi: jika sistem awalnya dieksitasi pure di mode 1, tetap di mode 1 selamanya (asalkan damping juga modal-orthogonal — Rayleigh damping).
💡 Verifikasi Numerik: Selalu cek orthogonality di code Anda setelah solve eigenvalue: print(Phi.T @ M @ Phi) harus diagonal (atau identity untuk mass-normalized). Jika ada off-diagonal terms > 1e-10, ada bug di code atau M/K matriks Anda. Catch silent errors paling efektif.
Bagian 05

Modal Matrix [Φ] & Coordinate Transformation:
Dari {x} Fisik ke {η} Modal

Modal matrix [Φ] adalah matriks N×N yang berisi semua N mode shape sebagai kolom. Matriks ini menjadi transformation matrix antara koordinat fisik {x} dan koordinat modal {η}. Inilah operasi sentral dari modal analysis: ubah masalah dari coupled physical coordinates ke decoupled modal coordinates.

Modal Matrix Construction & Transformation
[Φ] = [φ1 φ2 ... φN]  ·  {x} = [Φ]{η}  ·  {η} = [Φ]−1{x}
📊

Konstruksi Modal Matrix

Stack N mode shape (column vectors) menjadi matriks N×N. Kolom 1 = mode 1, kolom 2 = mode 2, dst. Python: Phi = eigvecs (output scipy.linalg.eigh sudah dalam format ini).

Phi[:, i] = mode shape i
🔄

Forward Transformation: x → η

Modal coordinates: {η} = [Φ]−1{x}. Untuk mass-normalized [Φ]: {η} = [Φ]ᵀ[M]{x} (lebih stable, karena [Φ]ᵀ[M] adalah inverse dari [Φ]). ηi = participation amplitude mode i.

η_i = φ_i^T M x (mass-normalized)
🔁

Inverse Transformation: η → x

Setelah solve dalam modal coordinates, kembali ke physical: {x} = [Φ]{η}. Setiap mode kontribusi ke physical displacement = φi·ηi. Total response = jumlah semua mode contributions (Bagian 06 superposition).

x = Σ φ_i · η_i
🎯

Diagonalization Effect

Modal transformation membuat matriks [M] dan [K] menjadi diagonal: [Φ]ᵀ[M][Φ] = diag(mi), [Φ]ᵀ[K][Φ] = diag(ki). Inilah essence decoupling — semua off-diagonal terms (yang menyebabkan coupling) menjadi nol.

Φ^T M Φ = M_diag (decoupled)
Transformasi Equation of Motion
Original (coupled): [M]{ẍ} + [K]{x} = {F(t)}
Substitute {x} = [Φ]{η} dan pre-multiply [Φ]ᵀ:
[Φ]ᵀ[M][Φ]{η̈} + [Φ]ᵀ[K][Φ]{η} = [Φ]ᵀ{F(t)}
Karena diagonalization: jadi N independent equations:
miη̈i + kiηi = fi(t)  untuk i = 1, 2, ..., N
Setiap modal equation = standard 1-DoF undamped oscillator dengan ωi² = ki/mi. Solve N independent ODEs (super easy), lalu transform balik ke physical via {x} = [Φ]{η}. Inilah magic of modal analysis.
💡 Modal Participation Factor: Untuk excitation tertentu (mis. seismic ground motion), modal participation factor Γi = φiᵀ[M]{r}/(φiᵀ[M]φi) dimana {r} = arah excitation. Mengukur seberapa kuat mode i ter-excite oleh load tersebut. Mode dengan Γ tinggi paling penting untuk dipertimbangkan dalam response analysis. Standard di seismic design.
Bagian 06

Modal Superposition Method:
Building Block Solusi Time-History MDoF

Modal Superposition adalah metode solusi MDoF time-history yang paling efisien: solve N independent modal equations (mudah, masing-masing 1-DoF), lalu superposisi kontribusi semua mode. Untuk damping Rayleigh (proportional damping), modal superposition exact. Untuk damping general, gunakan modal truncation untuk approximation.

Modal Superposition Equation
{x(t)} = Σ φi · ηi(t)  ·  ηi(t) dari independent modal ODE
📋

Workflow Step-by-Step

(1) Solve eigenvalue → ωi, φi. (2) Mass-normalize mode shapes. (3) Compute modal force fi(t) = φiᵀ{F(t)}. (4) Solve modal equation untuk ηi(t) (1-DoF, easy!). (5) Superpose: {x(t)} = Σ φi·ηi(t).

5 steps: eig → norm → modal F → solve → sum
🎚️

Modal Truncation (Mode Truncation)

Untuk MDoF besar (N=100+), tidak perlu hitung semua N mode. Truncate ke beberapa mode dominan saja (mis. mode 1-5). Akurasi 95% untuk loading dominasi low-frequency (gempa, angin). Komputasi 10-100× lebih cepat dari direct integration.

Truncate to k modes (k << N)
💧

Rayleigh Damping (Proportional)

Damping matrix [C] = α[M] + β[K]. Sifat: [C] juga diagonalized oleh [Φ] (modal damping). Tiap mode i punya damping ratio ζi independent. Modal equation dengan damping: η̈i + 2ζiωiη̇i + ωi²ηi = fi(t).

C = αM + βK (Rayleigh)
⚙️

Comparison vs Direct Integration

Direct integration (Newmark, Wilson-θ): solve coupled ODE [M]{ẍ}+[C]{ẋ}+[K]{x}={F} step by step. Pros: handle non-linear, non-proportional damping. Cons: lambat (O(N²) per step). Modal superposition: super cepat, tapi limited ke linear + Rayleigh damping. Pilih sesuai kebutuhan.

Modal: fast linear; Direct: slow general
Use Case Praktis: Seismic Response Building
Building 20 lantai (N=20 DoF). Loading: ground acceleration earthquake. Workflow:
1) Solve 20 eigenvalues, get ω1=1.5 Hz (first mode), ω2=4.5 Hz, dst.
2) Mass-normalize semua 20 mode shape.
3) Compute modal participation untuk arah horizontal: Γ1=1.4, Γ2=−0.5, Γ3=0.3, dst.
4) Truncate ke 3 modes pertama (Γi² > 5% total).
5) Solve 3 modal ODE dengan modal force = Γi·aground(t).
6) Superpose: x(t) = Σ Γi·φi·ηi(t).
Get inter-story drift, base shear, max acceleration per lantai. Standard practice di structural engineering software (ETABS, SAP2000).
💡 Akurasi Modal Truncation: Aturan praktis untuk seismic: pakai mode dengan kumulatif effective modal mass > 90%. Effective modal mass ke-i: Meff,i = (Γi)². Plot kumulatif vs mode count, stop saat > 90%. Biasanya 5-10 mode pertama cukup untuk building sampai 30 lantai. Tugas safety code: ASCE 7 mensyaratkan minimum 90% effective mass participation.
Bagian 07

Aplikasi MDoF:
Building, Bridges, Aerospace Structures

Modal analysis bukan sekadar math — adalah workhorse di structural engineering modern untuk gedung, jembatan, pesawat, dan kendaraan. Setiap field punya kasus klasik yang teach-by-example. Berikut beberapa contoh real-world.

Standard Industry Tools
ETABS (gedung) · SAP2000 (general structure) · Abaqus/Nastran (aerospace) · ANSYS (mechanical)
🏢

Tall Building (Burj Khalifa, Petronas)

828 m Burj Khalifa dimodelkan sebagai 100+ DoF. Mode 1 = sway lateral (T ≈ 11 sec), mode 2 = perpendicular sway, mode 3 = torsional. Wind load & seismic dianalisis dengan modal superposition. Tuned mass damper (Bagian 07 Modul-7) di-tune ke f1.

Burj Khalifa T_1 ≈ 11 sec (very low f)
🌉

Long-Span Bridge (Tacoma, Akashi)

Tacoma Narrows 1940: collapsed karena resonansi vortex shedding dengan mode torsional bridge. Lesson: modal analysis WAJIB include torsional modes. Akashi Kaikyo (1991 m): 200+ DoF analysis, modes 1-20 considered.

Tacoma: torsional flutter at ω_5
✈️

Aircraft Wing (Aeroelastic Flutter)

Wing pesawat = continuous structure di-discretize jadi 1000-100000 DoF FEM. Modal analysis identify mode bending vs torsion. Flutter: coupling unstable antara dua mode pada speed kritis. Wajib dihindari di certification.

Flutter at V_critical from modal coupling
🚗

Vehicle Body NVH Analysis

Body mobil = 10⁵-10⁶ DoF FEM. Modal analysis identify resonance frequencies yang dapat dirasakan penumpang sebagai noise/vibration. Optimization: shift mode frequencies away dari engine excitation (idle, cruising).

NVH: shift modes to f > 200 Hz
Workflow Industri Standar (Software-Based)
1) Geometry & Material: import 3D model dari CAD (SolidWorks, Catia).
2) Meshing: discretize ke finite elements (mis. 100K elements untuk car body). Tool: Hypermesh, ANSA.
3) Boundary Conditions: define support, load, contact.
4) Modal Solve: pilih jumlah modes (mis. 20-50 lowest), run solver (Nastran, Abaqus).
5) Post-Processing: visualize mode shapes 3D animation, frequency table, modal participation per direction.
6) Decision: redesign jika ada mode di range eksitasi (mis. shift natural frequency > 1.5× engine RPM range).
Tugas vibration engineer profesional: 60% running & interpreting these tools, 40% theoretical understanding (modul ini).
💡 Modal Testing (Experimental Modal Analysis): Selain analytical (FEM), modal analysis bisa dari eksperimen — impact hammer test atau shaker test. Measure response di banyak titik, extract mode shape dari ratio amplitude. Tools: PolyMAX, EFDD algorithm. Validate FEM model dengan eksperimen — ini standard di automotive & aerospace certification.
Bagian 08

Implementasi Python:
Modal Analysis 5-DoF Building Example

Empat cell Python berikut mengimplementasikan modal analysis lengkap untuk gedung 5 lantai (5-DoF lateral): setup matriks M & K (Cell 1), eigenvalue solve dengan scipy (Cell 2), modal transformation & orthogonality verification (Cell 3), modal superposition response untuk seismic ground motion (Cell 4).

Python · Cell 1 Setup 5-DoF Building (Mass & Stiffness Matrix) 
import numpy as np

# Building 5-lantai: tiap lantai 1 DoF lateral
# Mass per lantai: 1000 kg (uniform)
# Stiffness per lantai: 1.6e6 N/m (uniform)
N = 5
m_floor = 1000.0
k_floor = 1.6e6

# Mass matrix (diagonal, lumped mass)
M = np.eye(N) * m_floor

# Stiffness matrix (tridiagonal: shear building model)
K = np.zeros((N, N))
for i in range(N):
    K[i, i] += k_floor   # contribution from spring above
    if i < N - 1:
        K[i, i] += k_floor       # contribution from spring below
        K[i, i+1] = -k_floor
        K[i+1, i] = -k_floor

print(f"Mass matrix M ({N}×{N}):")
print(M)
print(f"
Stiffness matrix K ({N}×{N}):")
print(K)
print(f"
Trace(K) = {np.trace(K):.0f}, det(K) = {np.linalg.det(K):.2e}")
Python · Cell 2 Generalized Eigenvalue Solve dengan scipy.linalg.eigh 
from scipy.linalg import eigh

# Solve generalized eigenvalue: K φ = ω² M φ
# eigh returns mass-normalized eigenvectors automatically
eigvals, Phi = eigh(K, M)
omegas = np.sqrt(eigvals)
freqs = omegas / (2*np.pi)
periods = 2*np.pi / omegas

print(f"{'Mode':>5}{'ω (rad/s)':>15}{'f (Hz)':>12}{'T (sec)':>12}")
for i in range(N):
    print(f"{i+1:>5}{omegas[i]:>15.3f}{freqs[i]:>12.3f}{periods[i]:>12.3f}")

print(f"
Mode shapes (kolom = mode):")
print(np.round(Phi, 3))
Python · Cell 3 Verify Orthogonality & Plot Mode Shapes 
import matplotlib.pyplot as plt

# Verify M-orthogonality (mass-normalized: Phi^T M Phi = I)
M_modal = Phi.T @ M @ Phi
print("Phi^T M Phi (should be Identity):")
print(np.round(M_modal, 6))

K_modal = Phi.T @ K @ Phi
print("
Phi^T K Phi (should be diag(omega^2)):")
print(np.round(K_modal, 2))

# Plot mode shapes (each mode as bar from ground to top)
fig, axes = plt.subplots(1, N, figsize=(14, 4), sharey=True)

Python · Cell 4 Modal Superposition: Seismic Response Time-History 
from scipy.integrate import odeint

# Synthetic ground acceleration: 5 Hz earthquake-like signal
dt = 0.01
t = np.arange(0, 10, dt)
ag_t = 0.3 * np.sin(2*np.pi*5*t) * np.exp(-t/3)   # decaying earthquake (g)
ag_t *= 9.81   # convert to m/s²

# Modal participation factors (for horizontal ground motion)
r = np.ones(N)   # influence vector (uniform horizontal)
Gamma = Phi.T @ M @ r   # modal participation factor per mode

# Modal damping (assume Rayleigh: 5% for all modes)
zeta = 0.05

# Solve N independent modal ODEs: η_i'' + 2ζω_i η_i' + ω_i² η_i = -Γ_i a_g(t)
def modal_ode(y, t_val, omega_i, zeta_i, Gamma_i, ag_func):
    eta, deta = y
    ddeta = -2*zeta_i*omega_i*deta - omega_i**2*eta - Gamma_i*ag_func(t_val)
    return [deta, ddeta]

from scipy.interpolate import interp1d
ag_func = interp1d(t, ag_t, bounds_error=False, fill_value=0)

eta_all = np.zeros((N, len(t)))
for i in range(N):
    sol = odeint(modal_ode, [0, 0], t, args=(omegas[i], zeta, Gamma[i], ag_func))
    eta_all[i, :] = sol[:, 0]

# Superpose: x(t) = Phi @ eta(t)
x_all = Phi @ eta_all   # (N floors, len(t))

# Plot top floor displacement vs time
fig, ax = plt.subplots(figsize=(11, 5))
for i in range(N):
    ax.plot(t, x_all[i, :]*1000, label=f'Floor {i+1}', alpha=0.7)
ax.set_xlabel('Time (s)'); ax.set_ylabel('Displacement (mm)')
ax.set_title(f'Seismic Response — Modal Superposition (5-DoF Building)
Gamma factors: {np.round(Gamma, 2)}')
ax.legend(loc='upper right'); ax.grid(alpha=0.3)
plt.tight_layout(); plt.show()

print(f"Max displacement per floor (mm):")
for i in range(N):
    print(f"  Floor {i+1}: {np.max(np.abs(x_all[i, :]))*1000:.2f} mm")

Diagnosis: Mengapa eksitasi pada ω = 10 dan ω ≈ 17.32 menyebabkan amplitudo besar?

›

Hitung kedua frekuensi pribadi sistem dengan persamaan karakteristik det(K − ω²M) = 0 atau menggunakan numpy.linalg.eig. Bandingkan hasil dengan frekuensi eksitasi yang dilaporkan operator. Apa kesimpulan diagnosa Anda?

det(K − ω²M) = 0 ω₁² + ω₂² = trace(K) ω₁²·ω₂² = det(K)
QUICK CHECK — Frekuensi eksitasi 10 dan 17.32 rad/s adalah...
Frekuensi sembarang yang kebetulan menghasilkan amplitudo besar
Tepat sama dengan kedua frekuensi pribadi sistem (ω₁ dan ω₂) — ini RESONANSI
Frekuensi kerusakan komponen pegas saja (bukan sistem)
Frekuensi yang membuat damping minimum
✅ Tepat! det(K − ω²M) = (200 − ω²)² − 10000 = 0 → ω² = 100 dan 300. Maka ω₁ = 10 rad/s dan ω₂ = √300 ≈ 17.32 rad/s — persis frekuensi eksitasi yang menimbulkan amplitudo besar. Sistem mengalami RESONANSI di kedua frekuensi pribadinya. Ini juga bisa diverifikasi dengan trace identity: ω₁²+ω₂² = 100+300 = 400 = trace(K) ✓.
❌ Hitung dulu det(K − ω²M) = 0. Anda akan menemukan ω² = 100 dan 300, yang persis bersesuaian dengan ω yang dilaporkan operator. Kondisi ini disebut RESONANSI.

Solusi: Bagaimana mendesain vibration absorber untuk meredam mode 2 (yang merusak pegas)?

›

Eksitasi paling berbahaya adalah pada ω₂ ≈ 17.32 rad/s (yang merusak pegas tengah). Tugas Anda: rancang vibration absorber kecil (m_a, k_a) yang dipasang ke salah satu massa untuk meredam getaran di ω = 17.32 rad/s. Aturan tuning: √(k_a/m_a) = ω_drive. Jika m_a = 0.1 kg (10% massa utama), berapa k_a yang diperlukan? Bagaimana sistem 2 DoF asli berubah menjadi sistem 3 DoF setelah pemasangan absorber?

k_a = m_a · ω_drive² m_a ≈ 5–20% m_struktur
💬 Hint: untuk meredam ω_drive = 17.32 rad/s dengan m_a = 0.1 kg, perlu k_a = 0.1 × (17.32)² = 0.1 × 300 = 30 N/m. Setelah dipasang, sistem menjadi 3 DoF dengan tiga frekuensi pribadi baru — dua di antaranya berada di sekitar ω_drive (mengapit), satu lebih jauh. Trade-off: tidak ada lagi puncak resonansi tepat di ω = 17.32, tetapi muncul dua puncak baru yang lebih kecil di ω yang berdekatan.
